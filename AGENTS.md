## 关键约定

### 笔记整理

整理 `.md` 笔记时遵循：
- 大纲缩进转为 `##`/`###` 标题层级
- **保护用户已设置好的标题层级关系**：如果笔记中已经存在明确的 `#`/`##`/`###`/`####` 结构，默认只在原层级下整理内容，不主动重排、升降级、重编号标题；只有用户明确要求“重构标题层级/重新编号/统一标题体系”时才改标题层级。
- 图片统一加宽度：`![描述|宽度](url)`，原理图 400-500px，细节图 300px
- 能用 Markdown 表格表达的信息优先用表格，替代信息类图片
- 合并重复描述，删除冗余内容
- 残留的 LaTeX+纯文本公式拼贴修复为干净 LaTeX
- **优化笔记不是复制粘贴**：截图转文字或整理内容时，必须按信息重要程度合理使用 `**粗体**`、`==高亮==`、`*斜体*`、代码块、行内代码、列表、表格、callout、ASCII 框图等形式，让重点、易混点、命令、层级关系能一眼看出来。
- **最小级标题下优先用有序列表组织详细内容**：在当前最低层级标题下，如果有多个并列的操作、命令、概念或步骤，不要写成散落的短句小标题；优先写成 `1. 2. 3.` 有序列表。每个序号先给出概括性动作或结论（如 `1. 创建功能包`），再在该序号下面放对应的介绍、命令、参数解释、示例输出、注意事项等。
- **说明句后紧跟代码块时不额外空行**：如果一行文字以“：”或明显引出代码/目录/输出的说明句结尾，下一行可以直接接 fenced code block（如 ```bash、```text），不要为了通用 Markdown 习惯额外插入空行；Obsidian 可以正常渲染。
- **笔记内容优先用有序/无序列表组织**：知识点罗列、对比、步骤等先用列表表达，保持条理清晰、结构紧凑。仅当编号段落确实碎片化、无结构时，才转为表格或代码块
- 整理时顺手修正原文笔误（术语、拼写）
- 缺失的对照信息适当补充（如有序/无序容器对比表）
- Obsidian Callout 用于注意事项（`> [!warning]`）和补充说明（`> [!NOTE]`）
- 标题编号统一使用以下五层格式：
  - H1：中文数字，如 `一、标题`、`二、标题`。
  - H2：阿拉伯一级编号，如 `1.标题内容`、`2.标题内容`；在每个 H1 下从 `1.` 重新开始。
  - H3：阿拉伯二级编号，如 `1.1 标题内容`、`1.2 标题内容`；跟随所属 H2 编号。
  - H4：阿拉伯三级编号，如 `1.1.1 标题内容`、`1.1.2 标题内容`；跟随所属 H3 编号。
  - H5：括号编号，如 `(1) 标题内容`、`(2) 标题内容`；在每个 H4 下从 `(1)` 重新开始。
- 若笔记没有 H1，先在 YAML frontmatter 之后（无 frontmatter 时在文件开头）补入以文件名命名的 H1，再整理其他层级；不主动降低已有标题层级。
- Number Headings 插件不能精确表达本规则的中文 H1 和括号 H5；不得对采用本规则的笔记启用全局自动编号，以免将 H3/H4 改成带尾点或将 H5 改写为层级数字。
- 对于C/C++或者一些学习笔记中，如果能够用制表符画的框图来解释的更清楚的话，用框图来描述。或者文字+框图的方式描述。
- **新笔记与旧笔记重复时，新内容融合进旧笔记**：如果新增内容与已有笔记讲的是同一块知识，不要新建独立笔记，而是将新内容作为补充章节合并到旧笔记中对应位置

## 学习秘书团 Skill（独立项目）

`学习秘书团` 是本仓库内的一个独立 skill 项目，正渐进开发中。

### 当前状态（2026-05-30）

- **角色**：总秘书大人（总控）+ 问题官 + 知识官 + 复盘官
- **位置**：`.agents/skills/学习秘书团/SKILL.md`
- **已跑通闭环**：出题 → 对话考试 → 批改 → 错题入库 → 1-3-7 复盘
- **配套文件**：`复习追踪/`（题源.md、题库.md、错题记录.md、问题状态库.md）
- **设计文档**：`AI/agent秘书团构想.md`
- **近期新增**：评分键加速批改、出题/复盘并行、错题分级反馈、答题进度条、动态新题量、错选项解析

### 核心规则

- 题源驱动：用户维护 `复习追踪/题源.md`，`#重点` 笔记多出题 + 答对不删
- 答题方式：对话逐题考，即时判对错
- 批改标准：选择题选项匹配，填空题所有空全对才正确，简答 80% 语义准确
- 答对即删（#重点题除外），答错隔日考
- 到期错题全部必做，不设上限；新题量 = 8 - floor(M/5)，最少 3 道
- 出题时同步生成评分键，选择题/填空题直接匹配，简答题走知识官语义判断
- 错选项必须解析：选择、多选、判断的每个错误选项都要说明为什么错、正确的是什么

### 待升级方向

1. **自动定时复盘**：Hook 每天定时触发复盘官，不需手动
2. **Dataview 面板**：Obsidian 内可视化查看复习进度、弱项分析
3. **错题权重**：答错过的知识点在随机抽题时提高出现概率
4. **统计分析**：每周/每月正确率趋势、薄弱知识点排行
5. **题型扩展**：判断题（多选和代码题已实现）
6. **多题源分类**：按 C/C++/嵌入式 等分类出题
7. **语音问答**：支持语音输入答题

## C++

- 对于原理要讲的通俗易懂，合理使用比喻和例子解释说明。
- 同时补充拓展相关的小知识点常用法和注意点
- **命名空间显式限定**：使用 C++ 标准库或其他命名空间中的类型、函数、对象时，必须显式写出命名空间和作用域解析符 `::`，例如 `std::string`、`std::vector<int>`、`std::isupper(...)`；禁止依赖全局 `using namespace std;`。
- **函数代码输出要求**：给出 C/C++ 函数实现时，必须在函数定义前提供清晰的 API 注释，至少说明：功能与适用场景、各参数的含义/单位/有效范围、返回值或错误语义、是否修改输入数据、前置条件与边界情况。涉及嵌入式场景时，还应说明内存分配、中断上下文、并发/线程安全、溢出风险等相关约束；不适用的项目可明确标注“无”。
