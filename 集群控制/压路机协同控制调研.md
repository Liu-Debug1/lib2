# 压路机单机自动化与双机协同调研

> 调研目标：搞清楚"两台压路机怎么自动化协同工作"的实现方法，从单机自动化基础 → 双机通信/编队/控制 → 行业案例，逐层深入。

---

## 零、全景地图：一张图看懂全流程

> 在看任何资料之前，先把这张地图印在脑子里。后面每读一篇论文，你就问自己："它是解决哪一层的问题？"

### 0.1 单机：一台无人压路机怎么自己工作？

可以类比为**一个盲人在走路**，需要解决 4 个问题：

```
┌─────────────────────────────────────────────────────┐
│  第 1 层：感知与定位 —— "我在哪？周围有什么？"         │
│  技术栈：北斗 RTK + 毫米波雷达 + IMU + 摄像头 + 激光雷达 │
│  核心算法：卡尔曼滤波（多传感器融合）                    │
├─────────────────────────────────────────────────────┤
│  第 2 层：路径规划 —— "我要怎么走？"                    │
│  技术栈：贝塞尔曲线、A*、覆盖路径规划                    │
│  输入：施工图纸 / 工区边界                              │
│  输出：一条平滑的碾压轨迹                                │
├─────────────────────────────────────────────────────┤
│  第 3 层：运动控制 —— "方向盘打多少？油门踩多大？"        │
│  技术栈：Pure Pursuit / Stanley / PID / MPC          │
│  输入：目标轨迹 vs 当前位置的偏差                        │
│  输出：转向角指令、速度指令                            │
│  核心工作：把"路径偏差"翻译成"执行器指令"               │
├─────────────────────────────────────────────────────┤
│  第 4 层：执行器 —— 真正动手                            │
│  技术栈：液压机械臂 / 电磁阀 / 电机控制器                │
│  输入：转向角指令、速度指令                              │
│  输出：真实的转向、行走、振动、制动动作                   │
│                                    │
└─────────────────────────────────────────────────────┘
```

**完整数据流**：

```
北斗/雷达/IMU → 卡尔曼滤波融合 → 位置+姿态估计
    → 覆盖路径规划 → 目标轨迹点序列
        → PID/MPC 控制器 → 转向角、油门/制动指令
            → 液压电磁阀/机械臂 → 车辆真正动作
                → 传感器再采集新位置 → 闭环
```

### 0.2 集群：两台压路机怎么协同？

集群 = 单机 × N + **三层协作能力**：

```
┌──────────────────────────┐
│  任务调度层               │ ← 集群独有
│  谁碾第 1 道？谁碾第 2 道？│
│  技术栈：任务分配算法       │
├──────────────────────────┤
│  编队控制层               │ ← 集群独有
│  保持队形：间距对了没？     │
│  间距大了 → 加速           │
│  间距小了 → 减速           │
│  技术栈：Leader-Follower / │
│  一致性协议 / 人工势场 /   │
│  分布式 MPC               │
├──────────────────────────┤
│  通信交互层               │ ← 集群独有
│  两台机器交换位置+速度+状态 │
│  技术栈：ZigBee / V2X /   │
│  WiFi / LoRa             │
└──────────────────────────┘
            ↑
    这三层跑在每台机器的单机四层之上
            ↓
┌──────────────────────────┐
│  单机四层（每台机器独立）   │
│  感知→规划→控制→执行      │
└──────────────────────────┘
```

**集群核心问题**：两台机器各有各的感知和控制，如果没有通信和编队算法，它们就是**两台互不知晓的独立机器**，可能撞到一起或者碾压重叠/遗漏。

加了通信和编队控制之后，数据流变成：

```
机器 A：感知位置 → 发给机器 B
机器 B：感知位置 → 发给机器 A
    → 编队控制器：对比 A/B 相对位置 vs 期望队形
        → 算出各自需要调整的速度/方向
            → 各自单机控制器 → 执行器
```

## 一、单机自动化基础

### 1.1 压路机运动学与动力学建模

压路机一般是**铰接式结构**（前车体+后车体通过铰接点连接），跟乘用车的阿克曼转向不一样——铰接式转向靠液压油缸推动前后车体相对偏转。

| 论文/资料                           | 出处                      | 年份   | 要点                                                          |
| ------------------------------- | ----------------------- | ---- | ----------------------------------------------------------- |
| 铰接式无人压路机运动学建模与验证试验              | 《机械工程师》2025(11)         | 2025 | 多体运动学方程，实车圆弧循迹验证，位置误差 ≤ ±0.12m，航向误差 ≤ ±0.1°，比经典模型精度提升 3.08% |
| 无人驾驶单钢轮碾压机最优换道路径规划              | 《现代制造工程》(北大核心) 2024(6)  | 2024 | 五阶贝塞尔换道路径 + LSO 光谱优化器优化控制点，最大横向误差 -3.8%，平均油耗 -3.8%          |
| 铰接式无人压路机能耗最优换道路径规划              | 《机械设计与研究》(北大核心) 2024(3) | 2024 | INFO 算法优化五阶贝塞尔路径，实车验证节能效果                                   |
| 专利 CN121635459A：离散元-多体动力学耦合路径跟踪 | 同济大学                    | 2025 | DEM-MBD 耦合模型 + 滑移/沉降特征辨识 + MPC 控制，适应非结构化路面                  |

**关键认知**：铰接式压路机的运动学模型比轮式车辆更复杂，铰接角是核心状态量，建模时要考虑滑移和沉降（软基路面特有）。

### 1.2 线控化改造方案

**压路机没有原生线控系统**（跟乘用车不一样），需要后装改造。当前两条技术路线：

#### 路线一：外挂式机械臂（驾驶机器人）

- **代表厂商**：踏歌智行（与徐工合作）
- **原理**：在方向盘、油门/制动踏板、挡位上安装液压机械臂，由辅助控制器（STM32/ECU）驱动机械臂模拟人类驾驶员操作
- **组成**：
  - 转向液压机械臂 — 伸张/收缩控制方向盘
  - 挡位液压机械臂 — 控制前进/后退/驻车
  - 控制电门 — 并联振动开关，控制振动压实
- **专利**：CN113512924B（GPS + 毫米波雷达 + STM32 + 液压机械臂）
- **优点**：不改原车液压系统，安装快，适合存量设备改造
- **缺点**：机械臂有延迟和磨损，精度受限

#### 路线二：直接控制液压油路（内嵌式）

- **专利**：CN208183487U
- **原理**：电脑模块 + 输入模块（GPS/毫米波雷达/红外/图像）+ 输出模块（行走/制动/转向/振动的液压油缸和电磁阀），直接控制液压油路
- **优点**：响应快、精度高
- **缺点**：需要改液压系统，适配不同车型工作量大

> **趋势判断**：当前行业处于"传统液压 → 数字化液压控制"过渡期。存量改造用路线一，新车型逐步走向路线二。

### 1.3 单机路径跟踪控制

## 二、双机协同方案（核心）

### 2.1 通信架构

两台压路机怎么"对话"：

| 通信方式                 | 特点                 | 适用场景          |
| -------------------- | ------------------ | ------------- |
| **ZigBee**           | 低成本、低功耗、短距离（~100m） | 原型验证、小范围施工    |
| **LTE-V2X / NR-V2X** | 低延迟（ms 级）、远距离、高可靠  | 工程化应用         |
| **WiFi / 4G/5G**     | 通用性强，延迟取决于网络       | 云端调度 + 本地直连备份 |
| **LoRa**             | 超远距离（km 级）、低速率     | 广域监控、状态上报     |

**实际工程中**：三一、徐工用的是"本地通信链路 + 云平台"双层架构——本地 V2V 做实时协同，云端做全局调度和状态监控。

### 2.2 编队策略

两台压路机常见配合模式：

| 模式 | 描述 | 控制要点 |
|------|------|---------|
| **并排碾压 (Side-by-side)** | 两机并排、保持固定间距，同时前进/后退碾压相邻车道 | 横向间距保持 + 纵向速度同步 |
| **一前一后 (Leader-Follower)** | 领航机走预定轨迹，跟随机保持固定距离跟在后面 | 纵向距离保持 + 轨迹跟踪 |
| **先后入场 (One-by-One Entry)** | 第一台开始碾压→第二台延时入场→交替作业 | 入场时机 + 碰撞避免 |
| **编队碾压 (Platoon Driving)** | 多机成编队同时碾压，类似"摊铺机+压路机群"模式 | 全局路径规划 + 编队保持 |

> **两机场景最常用**：并排碾压（横向间距 0.2m 重叠量）和 Leader-Follower（纵向间距保持）。建议先搞懂这两种基本模式。

### 2.3 协同控制算法

| 方法                            | 原理                          | 论文/来源                                                            | 优缺点                   |
| ----------------------------- | --------------------------- | ---------------------------------------------------------------- | --------------------- |
| **Leader-Follower + PID**     | 领航者发位置/速度 → 跟随者 PID 调节间距和航向 | BIT 北理工 2024（兵工学报）：PID 纵向 + LQR 横向；北林 2024（WEVJ）：级联 PID（速度环+间距环） | **最简单、工程最常用**；领航者挂了全崩 |
| **Leader-Follower + MPC**     | 同上但控制律换成 MPC，能处理约束和预测       | Ma 2025（济南大学）：pilot-following + MPC，三台仿真稳定时间 ~7s                 | 精度高、能处理约束；计算量大        |
| **人工势场法 (APF)**               | 目标点给引力、障碍物/其他车给斥力           | Guo et al. 2025（中国工程机械学报）：APF + 微分齐次变换                           | 天然避障；局部极小值问题          |
| **虚拟结构法 (Virtual Structure)** | 定义一个虚拟刚体，各车跟踪刚体上的固定点        | Wei et al. 2025 (RAS)：虚拟结构 + 强化学习求解 HJB 方程，三台压路机坝面仿真             | 无单点故障；需要全局信息          |
| **强化学习 (RL)**                 | 神经网络在线学习最优编队策略              | Wei et al. 2025 (RAS)：critic NN 在线估计最优控制律                        | 适应复杂环境；训练成本高、可解释性差    |
| **一致性协议 (Consensus)**         | 各车只跟邻居通信，所有车收敛到共同状态         | 你现有笔记《控制算法学习》已覆盖                                                 | 分布式、可扩展；收敛速度取决于拓扑     |

**入门推荐路线**：
```
Leader-Follower + PID（2-3天跑通仿真）
  → Leader-Follower + MPC（加入运动学约束）
    → 人工势场法或虚拟结构（扩展到多机）
```

## 三、行业案例

### 3.1 三一重工

- **项目**：徐淮阜高速宿州段（2024 启动，持续 200+ 天，总里程 200+ km）
- **机群规模**：1 台摊铺机 + 3 台双钢轮 + 4 台胶轮，最多 15 台协同
- **关键技术**：融合感知定位 + 高精度路径跟踪 + 群控安全 + 全局协同调度
- **特色**：一键式施工（采集数据 → 自动建模 → 开启无人摊压），本地 + 云双平台
- **成效**：减少人员 60%，日效率 +20%，压实均匀性 +10%

### 3.2 徐工机械

- **累计里程**：智能化施工 4000+ km（常态化应用）
- **特色技术**：车载气象站（温度/风速自适应）、路面温度扫描系统
- **双机协同**：2024 上海宝马展演示完全无人压路机自主贴边 + 多车道双机协同
- **技术路线**：远程操控 → 自主感知 → 自主决策 → 全智能施工

## 四、与本专业的关联

### 4.1 线控底盘是执行层的核心

```
任务决策层 → 路径规划层 → 运动控制层 → 执行层（线控底盘）
                                              ↑
                                         本专业主战场
```

上层协同算法算出"该走什么轨迹、保持什么速度"，底盘执行层负责"能不能精准做到"。
