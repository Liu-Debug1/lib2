# 开源协同控制项目汇总

面向无人压路机协同控制的 GitHub 开源项目调研，按技术栈分类整理。

> [!warning]
> GitHub 上**没有**专门针对无人压路机协同控制的开源项目。施工现场机械自动化是高度封闭的工业领域，代码极少公开。以下项目按核心技术拆解后分别匹配，均可迁移适配。

---

## 一、核心技术拆解

```
无人压路机协同控制
├── ① 多机覆盖路径规划（核心难点）
│   └── 多台压路机协作覆盖整条路面，保证碾压遍数、搭接宽度
├── ② 编队/跟车控制
│   └── 前后压路机保持安全距离、错位编队
├── ③ 机间避碰
│   └── 低速重载，制动距离长，避碰策略不同
├── ④ 多机任务分配与调度
│   └── 路段划分、碾压遍数分配、动态重规划
└── ⑤ 车辆动力学控制
    └── 振动轮控制、速度控制、转向控制
```

---

## 二、覆盖路径规划（最核心）

> 压路机碾压 = 对矩形区域的多次全覆盖，与农业机械播种/收割路径**高度同构**（boustrophedon 往复式覆盖、转弯半径约束、多机任务分配、搭接宽度）。

| 项目                                                                                                 | 星标    | 语言         | 核心能力                                                                |
| -------------------------------------------------------------------------------------------------- | ----- | ---------- | ------------------------------------------------------------------- |
| **[Fields2Cover](https://github.com/Fields2Cover/Fields2Cover)**                                   | ~500+ | C++/Python | 农业全覆盖路径规划库，支持多车协同覆盖、非凸区域、障碍物绕行、Dubins/Reeds-Shepp 曲线。IEEE RA-L 2023 |
| **[CoveragePlanner](https://github.com/RicheyHuang/CoveragePlanner)**                              | ~238  | C++        | BCD 全覆盖 + TSP 路径优化，CGAL/OpenCV/Eigen                                |
| **[BCD-boustrophedon_path_planner](https://github.com/qiangsun89/BCD-boustrophedon_path_planner)** | —     | C++        | Boustrophedon 单元分解法全覆盖，适合矩形路面                                       |

### 关键概念映射

| 农业机械              | 压路机          |
| ----------------- | ------------ |
| Swath Width（作业幅宽） | 搭接宽度 / 碾压带宽度 |
| 覆盖次数              | 碾压遍数         |
| 地头转弯              | 路面两端掉头       |
| 多车分区作业            | 多机分段碾压       |

## 六、推荐实施路线

1. **先跑通 Fields2Cover**：把碾压场景输入 → 路面 = 矩形区域，多台压路机 = 多车，碾压遍数 = 覆盖次数，搭接宽度 = swath width。C++ 级别验证路径规划效果
2. **用 CARMA Platform 做编队控制**：已有成熟 Platooning 插件，底层 CACC 跟车逻辑 + V2V 通信框架可直接参考，替换为压路机动力学参数（质量、制动距离、最大速度）
3. **Gazebo 搭建多压路机仿真**：建压路机 URDF（车体 + 振动轮），ROS2 通信层，集成 Fields2Cover 路径 + CARMA 编队逻辑
