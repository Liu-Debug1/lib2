## 三、CACC 控制算法（核心）

### 3.1 纵向控制：PID 间隙保持

```
gap_error = desired_gap - actual_gap
speed_cmd = leader.cmdSpeed + PID(gap_error)
speed_cmd = clamp(speed_cmd, leader.speed ± max_adjustment)
speed_cmd = clamp(speed_cmd, prev_speed ± max_accel × dt)
```

具体实现在 `platooning_control_worker.cpp` 的 `generate_speed()` 方法：

1. 计算期望位置：`desiredPos = leaderPos - desiredGap`
2. 计算实际位置：`hostPos = leaderPos - actualGap`
3. PID 计算输出
4. 两层滤波器限制速度变化率

### 3.2 PID 参数

| 参数     | 默认值      | 说明           |
| ------ | -------- | ------------ |
| Kp     | 0.4      | 比例增益         |
| Ki     | 0.03     | 积分增益         |
| Kd     | 0.0      | 微分增益         |
| 积分限幅   | ±30.0    | 防止积分饱和       |
| 速度调整上限 | 15.0 m/s | 相对领导者速度的最大增量 |

### 3.3 领导者选择：APF 算法

**All-Predecessor Following**——动态选择编队中要跟随的车辆：

- 计算所有连续车辆对之间的时距
- 全部安全（1.6s~4.0s）→ 跟编队头车（效率优先）
- 任一时距超阈值 → 跟最近违规车辆（安全优先）
- 双层迟滞边界避免频繁切换

## 四、V2V 通信

### 消息类型

| 消息 | 方向 | 内容 | 频率 |
|------|------|------|------|
| `MobilityOperation` (STATUS) | 所有成员→全部 | CMDSPEED, SPEED, ECEF 位置 | 10 Hz |
| `MobilityOperation` (INFO) | 头车→全部 | 编队总长, 速度, 规模, ECEF 位置 | 5 Hz |
| `MobilityRequest` | 加入者→编队 | 请求加入参数 | 按需 |
| `MobilityResponse` | 编队→加入者 | ACK / NACK | 按需 |

### 坐标系统

使用 **ECEF 坐标**共享位置，接收方通过地图投影器转换到局部坐标系的 downtrack/crosstrack。

## 五、编队状态机

```
STANDBY ──→ LEADERWAITING ──→ LEADER
                  │                │
                  ▼                ▼
          CANDIDATEFOLLOWER    FOLLOWER
                  │                │
                  ▼                ▼
              FOLLOWER         STANDBY (离开)
```

支持三种加入方式：后方加入、前方加入（UCLA 扩展）、切入加入（UCLA 扩展）。

## 六、压路机场景的适用性

### 可直接参考的部分

1. **CACC 间隙保持**：PID 控制器 + 滤波器结构可直接用于压路机前后跟车
2. **V2V 通信框架**：MobilityOperation 消息模式可简化为压路机间状态广播
3. **三层插件架构**：战略(碾压策略)→战术(轨迹生成)→控制(执行)的分层设计

### 需要修改的部分

| CARMA 设计 | 压路机场景修改 |
|-----------|--------------|
| 高速 CACC (≥5 m/s) | 低速碾压 (1~3 m/s)，PID 参数需重新调 |
| 车道级定位 | 路面级定位，更简单 |
| 复杂交通场景 | 封闭施工场景，无交通参与者 |
| 编队行驶 (纵向队列) | 错位并排碾压，需横向偏移控制 |
| 纯追踪转向 | 铰接转向模型不同 |
| V2V 基于 DSRC | 可简化为 WiFi/4G/5G |

### 关键参数对照

| 参数 | CARMA 默认 | 压路机建议 |
|------|-----------|-----------|
| 跟车时距 | 1.2 s | 2~4 s (重载制动距离长) |
| 静止间隙 | 12 m | 3~5 m (低速，可更近) |
| 最大加速度 | 1.5 m/s² | 0.3~0.5 m/s² (大惯性) |
| 车辆长度 | 5.0 m | 5~7 m |
| 作业速度 | 2.2~30 m/s | 0.5~3 m/s |

## 七、评价与结论

### 优势

- **工程标准高**，USDOT 官方维护，代码规范
- **插件化设计**，三个编队插件职责清晰，可独立复用
- **V2V 通信成熟**，消息格式和协商协议经过实车验证
- **APF 领导跟随算法**理论扎实，有论文支撑
- **活跃维护**（6 天前还在更新）

### 不足

- **定位依赖强**：需要车道级地图 + ECEF 变换，压路机场景不需要
- **系统庞大**：~60 个 ROS 包，裁剪成本高
- **混合 ROS1/ROS2**：学习曲线陡
- **车辆模型固定**：Ackermann 转向，需改为铰接式

### 压路机适用度

| 维度 | 评分 | 说明 |
|------|:--:|------|
| CACC 跟车控制 | 可直接参考 PID 结构 |
| V2V 通信 | 协议模式可复用 |
| 编队状态机 | 加入/离开逻辑可迁移 |
| 横向控制(纯追踪) | 需改为铰接转向 |
| 直接部署 | 系统过重，需大量裁剪 |

**结论：CARMA 的核心价值在于 CACC 控制算法和 V2V 通信协议设计模式。不适合直接部署，但算法架构强烈推荐参考。**
