### Q021 | 来源：[[C语言基础]] | 创建：2026-05-23

**类型**：简答题

**题目**：在嵌入式开发中，为什么不能用 `==` 直接判断两个浮点数是否相等？应该如何正确比较？

**答案**：浮点数采用 IEEE 754 标准表示，多数十进制小数无法精确转换为二进制，存在舍入误差（如 0.1 在二进制中是无限循环小数）。多次运算后误差累积，两个"应该相等"的浮点数实际值可能差一个极小量。正确做法：使用容差比较 `fabs(a - b) < epsilon`，epsilon 根据精度需求设定（如 float 取 1e-6，double 取 1e-12）。

### Q022 | 来源：[[C++]] | 创建：2026-05-23 | #重点

**类型**：简答题

**题目**：在嵌入式开发中，什么场景应选 `struct`、什么场景应选 `class`？各举一个具体的工程例子，并说明选择理由。

**答案**：**选 struct 的场景**：纯数据聚合体或需要全公开访问的结构。例如寄存器映射——将 GPIO 外设寄存器建模为 struct，成员全 public，通过指针挂到硬件地址直接操作（`GPIOA->ODR |= (1<<8)`）。选择理由：寄存器本就应该公开访问，不需要封装。

**选 class 的场景**：需要保护内部状态、封装底层细节的模块。例如电机驱动——电流/速度等内部状态设为 private，通过 `setSpeed()`、`emergencyStop()` 等 public 接口对外暴露，非法的速度值在 setter 内部被拦截。选择理由：防止外部代码随意修改关键变量导致安全事故，封装后的接口更清晰。

核心原则：编译后零开销——class 的 `private` 在编译期检查，不增加运行时代码。

```c
int i = 0;      // 将i初始化为0
```

- 缺点：状态多时代码膨胀，修改需改动控制逻辑

**适用场景：** 状态数量固定且较少（3-5 个）用 switch-case，代码更直观；状态数量多（8+）或需要在运行时动态切换状态处理函数时用函数指针表；混合使用也很常见——外层 switch-case 分大类，内层函数指针表做子状态机。

### Q082 | 来源：[[C++]] | 创建：2026-05-31 | #重点

**类型**：简答题（分析层）

**题目**：分析以下代码，回答后面的问题：

```cpp
class Counter {
public:
    Counter(int start = 0) : count(start) {}
    Counter& increment() {     // 返回引用
        count++;
        return *this;
    }
    void print() const {       // const 成员函数
        cout << "当前值：" << count << endl;
    }
private:
    int count;
};
```

**(1)** 如果将 `increment()` 的返回类型从 `Counter&` 改为 `Counter`（值返回），调用 `c.increment().increment().print()` 时结果有什么不同？为什么？

**(2)** `print()` 函数末尾的 `const` 起什么作用？如果去掉这个 `const`，`const Counter c(5); c.print();` 能否编译通过？

**答案**：

**(1)** 返回 `Counter&`（引用）时，`c.increment()` 返回的是 `c` 自身的引用，两个连续的 `.increment()` 都作用在同一个对象 `c` 上，最终 `count` 变为 2，输出 `当前值：2`。
如果返回 `Counter`（值），`c.increment()` 返回的是 `c` 的临时副本，第二个 `.increment()` 修改的是副本而非 `c` 本身。更严重的是第一个 `.increment()` 返回的临时对象在表达式结束后销毁，整个过程与原始对象 `c` 无关——`c.count` 只自增了 1 次，最终 `print()` 输出 `当前值：1`。
链式调用的本质是"在同一个对象上连续操作"，必须返回引用。返回值的语义是"给你一个副本，随便玩"，与链式调用南辕北辙。

**(2)** `const` 修饰成员函数表示该函数承诺不会修改对象的状态（成员变量）。此时 `this` 指针的类型变为 `const Counter* const`——既不能改指向，也不能通过 `this` 修改任何成员值。
如果去掉 `const`，`const Counter c(5); c.print();` 会编译报错。C++ 规定 **const 对象只能调用 const 成员函数**。去掉了 `const` 的 `print()` 不再向编译器承诺"不修改对象"，编译器为了安全拒绝通过。这是 C++ const 正确性的底层保障机制。

**评分键**：
- 问题(1)：说明值返回导致链式调用作用在临时副本上 + 原对象 count 只增 1 次（得 50%）
- 问题(2)：const 成员函数不修改对象状态 + const 对象只能调用 const 成员函数（得 50%）
- 满分答案还需解释背后机制（this 类型变化、临时对象生命周期）

常见不足：
- 只说"结果不同"不说明具体原因 → 扣 50%
- 混淆 const 成员函数和 const 成员变量 → 扣 50%
- 认为 const 不是函数签名的一部分（const 确实是函数签名的一部分，影响重载决议）→ 扣 30%
