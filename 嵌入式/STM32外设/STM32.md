# STM32 开发笔记

## 1. 初识 STM32

### 1.1 什么是 STM32？

| 字母 | 含义 |
|:-----|:-----|
| **ST** | 意法半导体，推出 1000+ 款 Cortex-M 内核 MCU |
| **M** | MCU（区别于 MPU 的 Linux 方向） |
| **32** | 32 位处理器 |

![ST产品线全景\|500](https://api2.mubu.com/v3/document_image/bbe2c125-247b-4823-b3f5-3072679af0c0-28368310.jpg?)

### 1.2 命名规则

以 `STM32F103C8T6` 为例：

| 字段 | 含义 | 取值 |
|:-----|:-----|:-----|
| STM32 | 系列 | ARM Cortex-M 32 位 MCU |
| F | 类型 | F=通用型, L=低功耗, H=高性能 |
| 103 | 子系列 | 103=中密度性能型 |
| C | 引脚数 | C=48pin, R=64pin, V=100pin |
| 8 | Flash 容量 | 8=64KB, B=128KB, C=256KB |
| T | 封装 | T=LQFP, U=UFQFPN |
| 6 | 温度范围 | 6=-40~85°C, 7=-40~105°C |

![STM32命名规则\|500](https://api2.mubu.com/v3/document_image/528de554-7b14-41be-9a1b-a22430b798ed-28368310.jpg?)

> [!tip] 选型关注点
> 引脚数目 → 内存容量 → 封装 → 温度范围。性能由高到低，内存由大到小。

## 3. 最小系统

> [!info] 定义
> 最小系统是保证 MCU 正常工作的**最少电路组成**。

### 3.1 电源

- 多个 $V_{DD}$ / $V_{SS}$ 引脚**全部需要供电**，不可只接一路
- 芯片内部走线线宽小、阻抗大 → 外部电源走线直接供电
- $V_{BAT}$：RTC 后备供电区域，可接电池保持时钟

![电源引脚供电\|400](https://api2.mubu.com/v3/document_image/28368310_b10bde63-e6a9-4a14-c72a-2a88062b8aad.png?)

### 3.2 复位

NRST 引脚上拉至 $V_{DD}$，对地接按键构成复位电路。

![复位电路\|400](https://api2.mubu.com/v3/document_image/28368310_9e8fee37-c293-4705-8b46-c54a4821905d.png?)

### 3.3 启动配置

| BOOT0 | BOOT1 | 启动区域 |
|:------|:------|:---------|
| 0 | X | 主 Flash（正常运行） |
| 1 | 0 | 系统存储器（ISP 串口下载） |
| 1 | 1 | SRAM（调试用） |

默认 BOOT0、BOOT1 均下拉到 GND（从 Flash 启动）。

![BOOT配置\|400](https://api2.mubu.com/v3/document_image/28368310_bf8f3eeb-fd6d-479a-b706-676bac28c23b.png?)

### 3.4 晶振

- 8MHz 主晶振（HSE）
- 32.768kHz RTC 晶振（LSE，可选）

### 3.5 调试接口

| 接口 | 占用引脚 | 功能 |
|:-----|:---------|:-----|
| SWD | 2 | 调试 + 下载（推荐） |
| JTAG | 5 | 调试 + 下载 |
| USART 串口 | 2 | 仅下载 |

### 3.6 IO 分配原则

**优先分配特定外设 IO → 分配通用 IO → 最后微调。**

![最小系统引脚\|400](https://api2.mubu.com/v3/document_image/28368310_56b913d4-9f4d-4c84-c19d-6d78d4cbef7b.png?)

## 4. RCC 时钟树

> [!important] 核心原则
> **先时钟，后配置。** STM32 采用低功耗设计，所有外设时钟上电默认关闭。

### 4.1 需要开启时钟的外设

| 外设类型 | 何时需要 | 时钟总线 |
|:---------|:---------|:---------|
| GPIO 端口 | 使用任意 GPIO 引脚 | AHB / APB2 |
| 片上外设 (TIM/USART/SPI/I2C/ADC) | 使用对应外设模块 | APB1 或 APB2 |
| AFIO | 引脚重映射 / 调试端口配置 | APB2 |
| DMA 控制器 | 使用 DMA 数据搬运 | AHB |

### 4.2 不需要手动开启时钟的部件

| 部件 | 原因 |
|:-----|:-----|
| Cortex-M3 内核、Flash、SRAM | 系统初始化代码自动配置 |
| ADC 模拟部分、PWR、BKP | 由独立电源域控制 |
| SWD/JTAG 调试接口 | 默认开启（否则无法下载） |
| RTC / 备份域 | 通过 PWR + BKP 独立流程控制 |

> [!tip] 如何确定外设所在总线
> 查阅 STM32F10xxx 参考手册「存储器映射」章节，或直接看 `stm32f10x_rcc.h` 中的 `RCC_APB1Periph_xxx` / `RCC_APB2Periph_xxx` 列表。

## 5. GPIO

GPIO (General Purpose Input Output) 位于 **APB2** 总线上。每个 GPIO 端口 16 个引脚（GPIOA~GPIOx）。

### 5.1 输入驱动器

#### 保护二极管

| 情况 | 行为 |
|:-----|:-----|
| 引脚电压 > $V_{DD}$ | 上方二极管导通 → 钳位到 $V_{DD}$ |
| 引脚电压 < $V_{SS}$ | 下方二极管导通 → 钳位到 $V_{SS}$ |

#### 四种输入模式

| 模式 | 悬空时读数 | 特点 |
|:-----|:----------|:-----|
| **浮空输入** | 不确定 | 外部需已有上下拉 |
| **上拉输入** | 高电平 (1) | 内部 ~40kΩ 上拉，按键检测最常用 |
| **下拉输入** | 低电平 (0) | 内部 ~40kΩ 下拉，很少单独使用 |
| **模拟输入** | — | 直连 ADC，数字电路关闭，不能读电平/产生中断 |

![浮空输入\|300](https://api2.mubu.com/v3/document_image/3492b519-9869-467b-a9bb-c4b9eb890122-28368310.jpg?)
![上拉输入\|300](https://api2.mubu.com/v3/document_image/50a358a2-e0fd-4967-a19f-d00c11fded06-28368310.jpg?)
![模拟输入\|300](https://api2.mubu.com/v3/document_image/66e19648-49e0-42e3-99b3-ccf76b945e9e-28368310.jpg?)

![三种输入区别\|400](https://api2.mubu.com/v3/document_image/25e5ca1f-5e45-4b1f-ad91-89328f447490-28368310.jpg?)

![输入结构电路图\|400](https://api2.mubu.com/v3/document_image/d62221f6-203a-43c2-ad1b-ef8c0ab3f5b6-28368310.jpg?)

### 5.2 输出驱动器

#### 推挽输出 (Push-Pull)

内部有 P-MOS 和 N-MOS 各一个，**同一时间只有一个导通**：
- P-MOS 导通 → 输出 $V_{DD}$ (3.3V)
- N-MOS 导通 → 输出 $V_{SS}$ (GND)
- 驱动能力强（最大 ~20mA）

![推挽输出结构\|400](https://api2.mubu.com/v3/document_image/6b7e6a87-a8e9-45bd-bed6-df251e04ce30-28368310.jpg?)

**标准库操作**：
- `GPIO_SetBits(GPIOx, GPIO_Pin_x)` → 高电平
- `GPIO_ResetBits(GPIOx, GPIO_Pin_x)` → 低电平

![推挽输出波形\|400](https://api2.mubu.com/v3/document_image/719565c1-9528-49c3-b800-feae00d97198-28368310.jpg?)

#### 开漏输出 (Open-Drain)

只有 **N-MOS**，无 P-MOS：
- N-MOS 导通 → 输出低电平
- N-MOS 截止 → 高阻态（**不输出高电平，必须外接上拉电阻**）

![开漏输出结构\|400](https://api2.mubu.com/v3/document_image/b40e043d-9cd5-4474-8a7a-0d9bc7d3c1f3-28368310.jpg?)

**用途**：
- **电平转换**：STM32 只能输出 3.3V，外接上拉电阻到 5V 可驱动 5V 外设
- **线与功能**：任一设备输出低电平，总线即为低电平

![开漏线与功能\|400](https://api2.mubu.com/v3/document_image/b0727489-04ab-4eec-be9a-d3a67f385eea-28368310.jpg?)

#### 推挽 vs 开漏

| 对比 | 推挽 | 开漏 |
|:-----|:-----|:-----|
| 输出高电平 | 直接输出 $V_{DD}$ | 高阻态，需外接上拉 |
| 驱动能力 | 强（~20mA） | 由上拉电阻决定 |
| 适用场景 | 通用输出 | 总线、电平转换 |

![推挽vs开漏\|400](https://api2.mubu.com/v3/document_image/6943ff7e-ac7b-458f-be9a-c67ee28a041e-28368310.jpg?)

> [!tip] 选择原则
> 除了**必须使用开漏输出**的情况（I2C、电平转换），其他一律用**推挽输出**。

#### 复用输出

- **复用推挽输出 (AF_PP)**：引脚控制权交给内部外设（如 USART、PWM），推挽结构
- **复用开漏输出 (AF_OD)**：同上，开漏结构（如 I2C）

### 5.3 GPIO 模式汇总表

| 宏定义 | 模式 | 类型 | 电平特性 | 典型场景 |
|:-------|:-----|:-----|:---------|:---------|
| `GPIO_Mode_IN_FLOATING` | 浮空输入 | 输入 | 悬空不确定 | 外部已有上下拉的按键 |
| `GPIO_Mode_IPU` | 上拉输入 | 输入 | 悬空=高 | 独立按键（最常用） |
| `GPIO_Mode_IPD` | 下拉输入 | 输入 | 悬空=低 | 外部高有效开关 |
| `GPIO_Mode_AIN` | 模拟输入 | 输入 | 0~3.3V 连续 | ADC/DAC 采集 |
| `GPIO_Mode_Out_PP` | 推挽输出 | 通用输出 | 高=3.3V, 低=0V | LED、继电器、电机 IO |
| `GPIO_Mode_Out_OD` | 开漏输出 | 通用输出 | 低=0V, 高=高阻 | 总线、电平转换 |
| `GPIO_Mode_AF_PP` | 复用推挽 | 复用输出 | 外设自动控制 | USART TX、PWM、SPI |
| `GPIO_Mode_AF_OD` | 复用开漏 | 复用输出 | 外设自动控制 | I2C、SMBus |

### 5.4 重要注意事项

1. **开漏必须接上拉**：无论普通开漏还是复用开漏，需 4.7kΩ~10kΩ 上拉电阻到 3.3V
2. **模拟输入不能用数字功能**：配置后数字输入电路关闭，不能读电平/产生中断
3. **复用模式必须开外设时钟**：配置 AF_PP / AF_OD 后必须开启对应外设时钟
4. **推挽输出不能直接并联**：两个推挽输出连在一起 → 一端高、一端低 = 短路烧片
5. **5V 容忍引脚**：STM32F1 大部分引脚可接 5V 设备输入，但输出最高 3.3V

![GPIO引脚分组\|400](https://api2.mubu.com/v3/document_image/d33cce1e-2c13-4ff1-a002-9527da4ced7c-28368310.jpg?)

## 6. EXTI 外部中断

### 6.1 基本概念

| 概念 | 说明 |
|:-----|:-----|
| **中断** | 程序执行中触发特殊条件，转而执行其他程序 |
| **中断优先级** | 安排多个中断的缓急轻重 |
| **中断嵌套** | 执行中断时，被更高优先级中断抢先 |

### 6.2 EXTI 工作流程

```
IO(电平变化) → EXTI(申请中断) → AFIO(引脚选择) → NVIC(裁决) → CPU(执行中断函数)
```

![EXTI流程\|400](https://api2.mubu.com/v3/document_image/28368310_7e5035d3-c7ad-4dea-c556-c33888c9f736.png?)

**触发方式**：上升沿 / 下降沿 / 双边沿

![触发方式\|400](https://api2.mubu.com/v3/document_image/28368310_cd41941d-b082-4df0-a018-cf6c20cd3278.png?)

**适用场合**：接收不知何时变化的外部信号。

![EXTI应用场景\|725](https://api2.mubu.com/v3/document_image/28368310_63b31c05-aaab-4c38-8a5d-85c884531c93.png?)

> [!warning] 按键检测建议
> 按键检测用**定时器中断**触发（去抖动），不建议用 EXTI。

### 6.3 NVIC 中断嵌套

NVIC 负责 CPU 的中断分配。

| 优先级类型 | 作用 |
|:-----------|:-----|
| **抢占优先级** | 可中断正在响应的程序（中断嵌套） |
| **响应优先级** | 抢占相同时可插队，成为下一个响应者 |

> 值越小，优先级越高。

![NVIC优先级分组\|725](https://api2.mubu.com/v3/document_image/28368310_642f3186-96d4-47e5-b84d-47ac88002e5e.png?)

### 6.4 配置步骤

1. 配置 RCC 时钟（GPIO + AFIO）
2. 初始化 GPIO（上拉输入）
3. AFIO 选择中断引脚
4. EXTI 配置（触发方式等）
5. NVIC 配置（优先级分组建议放 `main()` 中 `while(1)` 之前）
6. 编写中断服务函数（函数名要与启动文件向量表一致）

## 8. TIM 定时器

### 8.1 资源概览

STM32F103C8T6 拥有 TIM1、TIM2、TIM3、TIM4。

- **16 位计数**：最大 65535
- **主频 72MHz**：72 个计数 = 1μs

### 8.2 基本定时器框图

![基本定时器框图\|500](https://api2.mubu.com/v3/document_image/28368310_0db1c178-f33b-4681-f8be-f1273c1c0617.png?)

| 组件 | 说明 |
|:-----|:-----|
| **预分频器 (PSC)** | 16 位，输出频率 = 输入频率 / (PSC+1) |
| **计数器 (CNT)** | 对时钟上升沿自增计数，16 位 |
| **自动重装载寄存器 (ARR)** | 存储目标值，CNT=ARR 时触发中断并自动清零 |

**计数器频率**：$f_{CK\_CNT} = f_{CK\_PSC} / (PSC + 1)$

### 8.3 通用定时器

#### 计数模式

| 模式 | 行为 |
|:-----|:-----|
| **向上计数** | 0 → ARR → 触发中断 → 归零 |
| **向下计数** | ARR → 0 → 触发中断 → 重置 |
| **中央对齐** | 0 → ARR（中断）→ 0（中断），对称波形 |

#### 时钟源

- **内部 72MHz**：最常用
- **外部时钟模式 1**（ETR 引脚 / ITR 级联 / CH1 / CH2）
- **外部时钟模式 2**（ETR 引脚，走 EXRF 通道）
- **编码器接口**：获取正交编码波形

![定时器结构图\|500](https://api2.mubu.com/v3/document_image/28368310_a8cfdc48-dc98-41db-e498-665d30613559.png?)

### 8.4 预分频器时序

![预分频器时序\|400](https://api2.mubu.com/v3/document_image/28368310_89a04e76-94c3-4dc5-a9b7-37833461772d.png?)

> [!note] 影子寄存器
> 预分频器有**缓冲机制**：PSC 值改变后，当前计数周期仍以旧值完成，下个周期才生效。
> 真正起作用的寄存器是**预分频缓冲器（影子寄存器）**。

### 8.5 计数器时序

![计数器\|400](https://api2.mubu.com/v3/document_image/28368310_96534efd-d2ce-4409-a9dc-8c13f4ec2fcf.png?)

**无预装影子寄存器**：ARR 改变立即生效——若 CNT 已超新 ARR，会先计到 0xFFFF 再归零，产生一次异常长周期。

![无缓冲时序\|400](https://api2.mubu.com/v3/document_image/28368310_4c3a233a-cefe-4675-cdda-a14dac4dec78.png?)

**有预装影子寄存器**：ARR 改变后，当前周期仍以旧 ARR 完成，下个周期才生效。

![有缓冲时序\|400](https://api2.mubu.com/v3/document_image/28368310_bd974a26-2283-4fc1-da53-d967620dde3b.png?)

### 8.6 定时器中断

#### 内部中断

- 配置 TIM + NVIC
- 更新频率：$$f_{update} = \frac{72\text{MHz}}{(PSC+1) \times (ARR+1)}$$
- 中断标志位需**手动清零**

#### 外部中断

- 配置 TIM + 对应 Pin + NVIC
- Pin 引脚脉冲作为时钟输入，CNT = ARR 时触发中断

### 8.7 资源分配

根据**引脚定义表**选择定时器通道。TIM 通道比较输出时，GPIO 模式须为**复用推挽输出 (AF_PP)**。

![定时器资源分配\|400](https://api2.mubu.com/v3/document_image/28368310_b5732411-4553-47e0-c9bd-afd637d80787.png?)

## 9. PWM 控制

### 9.1 基本概念

通过数字量控制等效出模拟量（调制脉冲宽度），常用于**电机控速**等领域。

| 参数 | 公式 |
|:-----|:-----|
| 频率 | $f = 1 / T_S$ |
| 占空比 | $Duty = T_{ON} / T_S$ |
| 分辨率 | $Reso = 1 / (ARR+1)$ |

> 占空比越大 → 等效越趋近高电平，保持线性关系。

![PWM参数示意\|400](https://api2.mubu.com/v3/document_image/28368310_ed72f47c-6159-4609-911c-f35788de3b46.png?)

### 9.2 PWM 基本结构

以 **TIM3_CH1、PWM1 模式、ARR=999、CCR=700** 为例：
- CNT 从 0 数到 699 → OCxREF 为**高电平**
- CNT 从 700 数到 999 → OCxREF 为**低电平**
- 结果：占空比 = 70% 的 PWM 波形

![PWM基本结构\|400](https://api2.mubu.com/v3/document_image/28368310_fbd37303-fe0d-41fe-81bd-6427960434e7.png?)

| 参数 | 公式 |
|:-----|:-----|
| PWM 频率 | $f_{PWM} = \frac{72\text{MHz}}{(PSC+1) \times (ARR+1)}$ |
| PWM 占空比 | $Duty = CCR / (ARR+1)$ |
| PWM 分辨率 | $Reso = 1 / (ARR+1)$，ARR 越大分辨率越高 |

> 一般取 1% 分辨率。1MHz = 1μs，1KHz = 1ms。

### 9.3 输出比较 (Output Compare)

通过比较 CNT 与 CCR 寄存器值，对输出电平进行**置 1、置 0 或翻转**，产生 PWM 波形。

- 每个高级/通用定时器有 **4 个输出比较通道**
- 高级定时器前 3 通道有**死区生成和互补输出**
- 默认使用 **PWM1 模式**（最常用）

## 10. 通信协议

### 10.1 通信基础概念

| 分类维度 | 类型 | 说明 | 常见协议 |
|:---------|:-----|:-----|:---------|
| **数据方向** | 全双工 | 收发同时，互不干扰 | UART、SPI、以太网 |
| | 半双工 | 同一时刻只能发或只能收 | I2C、单总线 |
| **时钟** | 同步 | 有独立时钟线，速度快 | SPI、I2C、CAN |
| | 异步 | 无时钟线，靠波特率约定 | UART |
| **信号形式** | 单端 | 1 信号线 + GND，易受干扰 | UART、SPI、I2C |
| | 差分 | 2 信号线，电压相反，抗干扰极强 | RS485、CAN、USB |

#### 常见电平标准

| 标准 | 逻辑 1 | 逻辑 0 | 应用 |
|:-----|:-------|:-------|:-----|
| TTL | 3.3V / 5V | 0V | 单片机直连 |
| RS232 | -3~-15V | +3~+15V | 电脑串口 |
| RS485 | 两线压差 +2~+6V | 两线压差 -2~-6V | 工业长距离 |

### 10.2 串口通信 (USART)

#### 简介

串口成本低、通信简单，可连接单片机↔单片机、单片机↔电脑、单片机↔各类模块。

**常用模块**：USB 转串口 (CH340)、陀螺仪、蓝牙串口 (HC-05)。

#### 硬件电路

- 简单串口：TX（发送）、RX（接收）
- TX ↔ RX **交叉连接**
- 单向传输只需 1 根线
- **必须共 GND**
- 电平不匹配时需电平转换芯片

![串口硬件电路\|400](https://api2.mubu.com/v3/document_image/28368310_16d77a64-3ff9-4bd0-d6a0-314dce595740.png?)

#### 数据帧格式

| 字段 | 说明 |
|:-----|:-----|
| **起始位** | 1 位，低电平，标志帧开始 |
| **数据位** | 5~9 位（通常 8 位），低位先行 |
| **校验位** | 可选：无/奇/偶校验 |
| **停止位** | 0.5/1/1.5/2 位，高电平 |

**波特率** = 码元/秒，二进制调制时与比特率相同。

#### STM32 USART 工作原理

USART 是 STM32 内部硬件外设，可自动生成/解析数据帧时序：
- 自带波特率发生器（最高 4.5Mbps）
- 可配置数据位长（8/9）、停止位长
- 支持同步模式、DMA、硬件流控制

**STM32F103C8T6 资源**：USART1 (APB2)、USART2 (APB1)、USART3 (APB1)

**初始化**：RCC + GPIO + USART + NVIC
- TX：复用推挽输出
- RX：浮空输入或上拉输入（保证空闲高电平）

**发送流程**：写 DR → 硬件自动发送 → TXE 标志置位 → 清除标志
**接收流程**：RXNE 标志置位 → 读 DR → 清除标志

#### 数据包接收

| 方式 | 适用场景 |
|:-----|:---------|
| **HEX 数据包** | 数据传输，固定包长避免与包头重复 |
| **文本数据包** | 人机交互，字符编码后不易与包头包尾冲突 |

![HEX数据包\|400](https://api2.mubu.com/v3/document_image/28368310_b791da05-a129-4d2a-b941-886a262fea07.png?)
![HEX数据包2\|400](https://api2.mubu.com/v3/document_image/28368310_ce770472-6a43-4528-e6d3-4698344b0114.png?)

| 事项 | 说明 |
|:-----|:-----|
| 文本 vs HEX | 本质都是二进制，只是显示/发送格式不同 |
| 发送字符串 | 用双引号 `"abc"` |
| 发送十六进制 | 用 `0x` 前缀 |

### 10.3 I2C 通信

#### 简介

| 特性 | 说明 |
|:-----|:-----|
| 通信线 | SCL（时钟）+ SDA（数据），2 线 |
| 模式 | 同步、半双工、带数据应答 |
| 拓扑 | 一主多从 / 多主多从 |

#### 硬件电路

- **SCL 和 SDA 均配置为开漏输出**
- 所有设备 SCL 连一起，SDA 连一起
- SCL 和 SDA 各加 **4.7kΩ 上拉电阻**（模块通常已自带）
- 利用线与：任一设备输出低 → 总线为低

![I2C硬件电路\|400](https://api2.mubu.com/v3/document_image/28368310_0b7d82ba-90bb-47cf-882a-0afe0d05aebe.png?)

#### 时序规则

| 条件 | SCL 状态 | SDA 变化 |
|:-----|:---------|:---------|
| **起始** | 高电平 | 高 → 低 |
| **终止** | 高电平 | 低 → 高 |
| **数据发送** | 低电平期间放数据 | 高电平期间**不允许变化**（读取时刻） |

![I2C起始终止\|400](https://api2.mubu.com/v3/document_image/28368310_11e5024f-d41e-45f5-f1fc-2604ce54936f.png?)
![I2C主机发送1字节\|400](https://api2.mubu.com/v3/document_image/28368310_596b269a-aac5-4a20-d86d-3612012ec6c6.png?)
![I2C主机接收1字节\|400](https://api2.mubu.com/v3/document_image/28368310_54f19379-21c1-4ecf-e3bb-bc2133b6d9b8.png?)

#### 设备地址

I2C 标准支持 7 位和 10 位地址：
- **7 位地址左移 1 位** → 写入时 +0（W），读取时 +1（R）
- 同型号芯片通过改变地址可变部分区分

#### 读写流程

| 操作 | 流程 |
|:-----|:-----|
| **指定地址写** | START → 地址+W → ACK → 寄存器号 → ACK → 数据 → ACK → STOP |
| **指定地址读** | START → 地址+W → ACK → 寄存器号 → ACK → Repeated START → 地址+R → ACK → 读数据 → NACK → STOP |

#### 软件 I2C vs 硬件 I2C

| 对比维度 | 硬件 I2C | 软件 I2C |
|:---------|:---------|:---------|
| 稳定性 | 一般（STM32F1 有 bug） | **极高**（代码完全可控） |
| 速度 | 快（100~400kHz） | 慢（~100kHz 以下） |
| 引脚灵活性 | 差（固定引脚） | **极高**（任意 GPIO） |
| CPU 占用 | 低 | 高 |
| 移植性 | 差 | **极高** |
| 调试难度 | 高 | **低** |

> [!tip] 本项目的选择
> 项目中使用**软件 I2C**（PB8/PB9 接 OLED），硬件 I2C1（PB6/PB7）仅挂载 INA219×2。

### 11.1 按键原理

- 按下导通，松开断开
- 按下/松开时产生**波形抖动**（5~10ms）

![按键抖动\|300](https://api2.mubu.com/v3/document_image/28368310_0eaefd13-baa1-4df5-b651-d8068048e6b5.png?)

**用法**：利用上拉输入（默认高电平），按键未按=1，按下=0。

![按键接线\|300](https://api2.mubu.com/v3/document_image/28368310_e98104e1-dc80-474f-bc5e-fe4aab375c0f.png?)

### 11.2 阻塞式 vs 非阻塞式

| 方式 | 特点 | 实现 |
|:-----|:-----|:-----|
| **阻塞式** | CPU 等待延时，暂停其他操作 | `delay_ms()` 消抖 |
| **非阻塞式** | CPU 不等待，定时扫描 | 定时器中断 + 状态机 |

### 11.3 全功能按键——状态机

通过标志位记录按键状态，实现对**短按、长按、双击**等操作的完整识别：

| 操作 | 代码 |
|:-----|:-----|
| 强制置 1 | `Flag \|= 0x01` |
| 强制置 0 | `Flag &= ~0x01` |
| 判断末位 | `if (Flag & 0x01)` |

## 12. 状态机设计

### 12.1 核心思想

把复杂的分步操作拆成一个个**状态**，每次主循环只处理当前状态，通过条件判断跳转到下一个状态——**完全非阻塞**，不耽误主循环。

### 12.2 设计三步法

1. **拆分状态**：按时间顺序/操作步骤拆成独立状态，每个状态只做一件事
2. **设置转移条件**：电平变化、数据包接收标志、定时超时等
3. **Switch-case 实现**：每个 case 一个状态

### 12.3 状态机的位置

| 位置 | 是否允许 | 原因 |
|:-----|:---------|:-----|
| ❌ 定时器中断 ISR | 不允许 | ISR 必须快进快出 |
| ❌ 外部中断 ISR | 不允许 | 同上 |
| ✅ `main()` 的 `while(1)` | **唯一正确** | 轮询调用，不阻塞 |
| ✅ 模块 .c 文件 | 正确 | 封装在对应文件中 |

### 12.4 注意事项

| 要点 | 说明 |
|:-----|:-----|
| 标记变量加 `volatile` | 防止编译器优化，主循环检测不到 |
| 检测标记后立即清零 | 避免重复执行 |
| 中断只做标记 | 保持中断快进快出 |
| 不同事件用不同标记 | 逻辑清晰，不混用 |
| 状态机封装在模块 .c 中 | 保持模块化 |
