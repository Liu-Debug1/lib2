# 代码规范

> 来源：团队代码规范文档，适用于嵌入式 C/C++ 项目（线控底盘方向）。

---

## 一、变量命名规范

采用**三段式大驼峰**命名：`(1)_(2)_(3)`

| 段位  | 含义        | 说明                                        |
| --- | --------- | ----------------------------------------- |
| (1) | 作用范围      | `g` 全局(整个终端所有.c可用)、`l` 局部(函数内)、`m` 成员(类中) |
| (2) | 所属模块      | 模块简写（见表1）                                 |
| (3) | 具体含义 + 类型 | 大驼峰含义 + 下划线 + 类型后缀                        |


**类型后缀：**

| 后缀 | 含义 |
|------|------|
| `b` | bool |
| `u8`/`u16`/`u32` | 无符号 8/16/32 位 |
| `s8`/`s16`/`s32` | 有符号 8/16/32 位 |
| `db` | double |
| `p` + 类型 | 指针，如 `ps8` 表示 s8 数组指针 |

**示例：**
```c
gLC_VehPosF_db    // 全局-定位-车辆位置滤波(double)
lSteerAngCmdFf    // 局部-转向角指令前馈
```

### 表1：模块简写

| 模块名称 | 简写 |
|---------|------|
| 定位 | LC |
| 规划 | PLN |

## 二、函数命名规范

采用**两段式小驼峰**命名：`(1)_(2)`

| 段位 | 含义 | 说明 |
|------|------|------|
| (1) | 作用范围 | `XXX` 模块名（整个终端可用）、`m` 类成员 |
| (2) | 具体含义 | 小驼峰，动词开头 |

**动词开头规范：**

| 前缀 | 含义 | 示例 |
|------|------|------|
| `get...()` | 获取一个值 | `getUserById()` |
| `set...()` | 设置一个属性 | `setBackgroundColor()` |
| `is...()` | 判断是否成立 | `isFileExist()` |
| `find...()` | 查找特定元素 | `findElement()` |
| `calc...()` | 计算 | `calcDistance()` |
| `update...()` | 更新状态 | `updatePosition()` |
| `init...()` | 初始化 | `initMotor()` |
| `process...()` | 处理 | `processPayment()` |

> [!note]
> 单词之间不使用下划线，直接连写，首字母小写。

## 三、类的命名和定义

| 类别 | 命名方式 | 示例 |
|------|---------|------|
| 类的声明 | 单段式大驼峰 | `UserProfile`、`PaymentService` |
| 类的实例 | 两段式大驼峰 `(2)_(3)`，其中 (3) 固定为 `C` | `RollerPlanner_C` |
| 类中函数 | 严格参照函数命名规范 | `initXxx()` |

## 四、工程的要求

**工程定义**：一个终端内可单独运行的多个 `.c` 和 `.h` 文件组成。

- 所有文件放在同一个文件夹下
- 每个工程只允许有一个 `main` 函数
- 工程间变量交互只允许通过**消息**
- 工程间时序无法严格保证，需要控制多个功能块时序时，必须放在同一个工程下

**main.c 结构：**
```text
main 函数只包含两部分：
  1. 初始化
  2. 时序控制
其余代码不放在 main.c 中。
```

## 五、注释规范

| 注释目标    | 规范                          |
| ------- | --------------------------- |
| BUG 修改  | 在修改处标注 BUG 编号，Git push 时也标注 |
| 变量      | 单行注释 `//`                   |
| 函数      | 多行注释 `/* */`，放在函数声明开头       |
| 函数内代码结构 | 使用 `// #1.1` 方式标注结构层级       |

**函数注释示例：**
```c
/*
 * 功能: 比例阀的PID控制
 * 定义: WLT, 2020.06.11
 * 修改: WLT, 2020.06.11
 */
```

## 六、宏定义和头文件

### 使用宏定义的场合

- 模式切换
- 大量被重复调用的同一个参数

### 命名规则

全大写 + 下划线分割：
```c
#define MAX_BUFFER_SIZE 1024
```

### 模式切换示例

```c
#define PLT_VEHICLE     0  // 实车测试
#define PLT_SIMULATION  1  // 仿真
#define PLT_MODE        PLT_VEHICLE
```

### h 文件主要包含内容

- 大量相同的 `.h` 包含
- 宏定义
- 函数的声明

> [!warning]
> 宏定义只在一个 `.h` 文件中定义，需要使用 `include` 该文件即可，避免重复定义。

## 七、代码主体结构

采用**顺序结构**：

```
初始化 → 主循环 → 后处理
```

主循环内部同样采用顺序结构。如必要采用分支结构时：
- 从同一个起点开始
- 汇总到同一个终点
- 随后继续顺序结构

## 八、其他规范

- **数据录**：统一在一个文件录 `data_log`
- **架构先行**：规定好架构（文件名、目录、各工程功能），只在架构里面修改相应代码
- **头文件统一**：放在统一一个头文件工程里，提高可移植性
- **全程序时间统一**：设置一个单独函数算时间，全局代码都调用这个时间
- **主函数频率**：正常 10Hz，仿真时通过宏定义改为 100 倍
- **时钟**：在主函数里算时钟，不用 ROS 的时钟。程序一次运行 0.01s，时钟每调用一次 +0.01s
- **include 整理**：把所有基础库文件放在一个 `.h` 里，其他 `.c` 只需 include 这个共性 `.h` + 个别独特 `.h`
- **文件组织**：消息类型单独放在消息的 cpp 里，纯公式计算放在计算的 cpp 里
