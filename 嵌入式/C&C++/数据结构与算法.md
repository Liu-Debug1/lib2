# 一、数据结构与算法

程序 = 数据结构 + 算法

## 1.绪论

### 1.1 基本概念和术语

**数据项 < 数据元素 < 数据**

- **数据**：能够输入和被计算机存储的各种符号的集合
  - 数字型
  - 非数字型
- **数据元素**：数据的基本单元，在计算机中作为一个整体进行考虑和处理。也称作元素、记录、结点、顶点
- **数据项**：构成数据元素的、不可分割的最小单位
- **数据对象**：性质相同的数据元素的集合，是数据的一个子集

**数据结构**

数据元素相互之间的关系称为结构。

| 分类 | 说明 |
|------|------|
| 逻辑结构 | 与计算机无关，数据元素之间的关系 |
| 物理/存储结构 | 数据元素在计算机中的存储方式 |
| 运算与实现 | 对数据元素的操作 |

**逻辑结构划分（一）**：

1. **线性结构**：1:1 关系
2. **非线性结构**：1:n 关系、n:n 关系

**逻辑结构划分（二）**：

![逻辑结构划分|400](https://api2.mubu.com/v3/document_image/52bf0d7b-7ed4-41af-9a8f-5feee20c77da-28368310.jpg)

**物理/存储结构**：

| 类型 | 说明 |
|------|------|
| 顺序存储结构 | 用连续的存储单元依次存储数据元素，逻辑关系由存储位置（下标）表示 |
| 索引存储结构 | 通过索引表记录元素位置 |
| 链式存储结构 | 通过指针链接各元素 |

### 1.2 数据类型和抽象数据类型

**数据类型**：值的集合 + 值的一组操作

## 2.顺序表与 List ADT

### 2.1 顺序表基本操作

- 初始化
- 获取指定位置元素
- 插入元素
- 删除元素
- 获取长度

### 2.2 List ADT

- 列表为空时大小为 0
- 可插入、可移动
- 容量可知
- 可查阅修改（特定位置）
- 可指定元素类型

## 3.链表

链表节点结构：数据区 + 指针区（存储下一个数据的首地址）

### 3.1 链表 vs 数组

| 对比维度 | 数组 | 链表 |
|----------|------|------|
| 访问元素 | O(1) | O(n) |
| 插入（开头） | O(n) | O(1) |
| 插入（结尾） | O(1) 或 O(n) | O(n) |
| 插入（指定位置） | O(n) | O(n) |
| 内存使用 | 需连续空间，扩容需整体拷贝 | 灵活使用内存，无空间浪费 |
| 使用简单度 | 在 C 中更简单 | 较复杂 |

### 3.2 链表在 C 中的实现

- **单链表**：`| Data | next |`
- 头部插入
- 任意位置插入
- 任意位置删除
- 尾部插入（练习）
- **遍历**：顺序遍历时迭代的效率高于递归
- **双向链表**：`| prev | Data | next |`

> [!warning]
> 访问堆的唯一方法就是通过指针。

## 4.栈

栈（Stack）是一种列表，只能从顶部（栈顶）进行删除或插入。

### 4.1 基本操作

后进先出（LIFO），时间复杂度均为 O(1)：

| 操作 | 说明 |
|------|------|
| `push()` | 向栈顶插入一个元素 |
| `pop()` | 从栈顶移出一个元素，一般返回移出元素的值 |
| `top()` | 返回栈顶元素 |
| `isEmpty()` | 判断栈是否为空 |

> [!NOTE] 动态扩容
> 当前数组溢出后，拷贝到新的 2 倍空间数组。

### 4.2 应用场景

1. 实现递归
2. 编辑器的撤回操作（Ctrl+Z）

### 4.3 栈在 C 中的实现

相对数组无需担心容量不够，直到存储耗尽。

**经典问题**：

**（1）括号匹配**：用栈判断 `{ }`、`( )`、`[ ]` 是否匹配。

**（2）表达式求值**：用栈处理操作符与操作数的运算。

优先级：括号 > 次幂 > 乘除 > 加减，同级自左向右。

| 表达式类型 | 扫描方向 |
|------------|----------|
| 前缀表达式 | 自右向左 |
| 后缀表达式 | 自左向右（时间和空间代价最小） |

> 前缀和后缀求值必须遵循 `<operand> <operand> <operator>` 的逻辑，进栈运算。

**中缀转前缀/后缀**：操作数顺序永远不变，操作符顺序可能改变。自左向右解析时，若操作符进栈前有更高优先级的操作符，则弹出该操作符。

## 5.队列

队列（Queue）是一种列表，只能从队尾（rear）插入，从头部（front）删除。

先进先出（FIFO）。

### 5.1 基本操作

时间复杂度均为 O(1)：

| 操作 | 说明 |
|------|------|
| `Enqueue(x)` / `Push(x)` | 向队尾插入，`rear` 位置 +1 |
| `Dequeue()` / `Pop()` | 删除队头元素并返回其值 |
| `Front()` / `Peek()` | 查看队头元素的值 |
| `IsEmpty()` | 判断队列是否为空 |

**循环队列**：

- 判满：`if ((rear + 1) % N == front)` → 队列已满
- 入队更新 rear：`rear = (rear + 1) % N`
- 判空：`if (front == -1 && rear == -1)` → 队列为空
- 特殊情况：只有一个元素时 front 和 rear 在同一位置，删除后需将 front 和 rear 都置为 -1

## 6.树

### 6.1 基本概念

| 概念 | 定义 |
|------|------|
| Depth（深度） | 从根到节点 X 的路径长度，根的深度为 0 |
| Height（高度） | 从节点 X 到最远叶子的路径长度（路径单位 = 1） |

- 空树的高度 = -1
- 一个节点的树的高度 = 0
- 操作成本直接取决于节点高度 h

### 6.2 二叉树

节点结构：`| Left | Data | Right |`，每个节点最多两个子节点。

**分类**：

| 类型 | 定义 |
|------|------|
| 严格二叉树 | 每个节点只能有 0 个或 2 个孩子 |
| 完美二叉树 | 每层每个节点都有 2 个子节点，没有空位 |
| 完全二叉树 | 最底层除外，每层都满，底层必须向左靠齐 |
| 二叉搜索树 | 左子树值 ≤ 节点值 < 右子树值 |
| 平衡二叉树 | 左右子树高度差受限 |

**节点数量关系**：

![节点数量公式|400](https://api2.mubu.com/v3/document_image/92117687-f19b-46e0-a764-a8ae22fb0c1e-28368310.jpg)

- 第 $i$ 层最大结点数：$2^i$
- 完美二叉树高度：$h = \log_{2}(n+1) - 1$
- 完全二叉树高度：$h = \lfloor \log_{2}n \rfloor$（向下取整）

### 6.3 二叉树的实现

1. **动态创建节点**
2. **利用数组**（适用于完全二叉树）：

![数组实现二叉树|400](https://api2.mubu.com/v3/document_image/0d27e988-9c01-4484-b3c9-b60e630d042e-28368310.jpg)

- 节点 $i$ 的左子叶索引：$2i + 1$
- 节点 $i$ 的右子叶索引：$2i + 2$

### 6.4 二叉搜索树（BST）

排序后的数组用二分查找成本很小，但每次插入/删除后需重新排序：

![插入删除代价|400](https://api2.mubu.com/v3/document_image/c2d84a71-fb87-42fd-818a-989bdd73ffbc-28368310.jpg)

**定义**：每个节点的左子节点值 $\leqslant$ 该节点，右子节点值 > 该节点。

![二叉搜索树定义|400](https://api2.mubu.com/v3/document_image/2da23310-0e60-4c9b-8f20-1a508001ab7a-28368310.jpg)

> [!NOTE]
> - 递归在处理树时会一直使用
> - Heap（堆）在执行中内存动态分配；Stack（栈）和其他变量在程序执行时就已确定大小，无法改变

**常用操作**：

- Find the Max and Min
- Find the Height of the Tree
- 判断是否为二叉搜索树（时间复杂度 O(n)）

### 6.5 遍历树

#### 6.5.1 广度优先（BFS）

![广度优先遍历|400](https://api2.mubu.com/v3/document_image/1c9d3663-f80d-4a44-b831-bef9c38931af-28368310.jpg)

优先遍历同一深度的节点，利用队列（FIFO）+ 指针。

- 时间复杂度：O(n)
- 空间复杂度：最好 O(1)（单支树），最差 O(n)（完美二叉树）

#### 6.5.2 深度优先（DFS）

![深度优先遍历|400](https://api2.mubu.com/v3/document_image/d54ab08a-4c3d-4bcb-9b77-79828eb4c63a-28368310.jpg)

![深度优先遍历2|400](https://api2.mubu.com/v3/document_image/d7ae7baa-5861-48a1-89b0-b57c96855731-28368310.jpg)

使用递归，按三种顺序遍历：

| 遍历方式 | 顺序 |
|----------|------|
| 前序遍历 | 根 → 左 → 右 |
| 中序遍历 | 左 → 根 → 右（BST 中按从小到大排列） |
| 后序遍历 | 左 → 右 → 根 |

- 时间复杂度：O(n)
- 空间复杂度：最好 $O(\log_{2}n)$（平衡树），最差 O(n)（单支树）

### 6.6 删除二叉搜索树节点

| 情况 | 处理方式 |
|------|----------|
| 无子节点 | 直接释放该节点 |
| 一个子节点 | 断开连接，父节点指向子节点，释放该节点 |
| 两个子节点 | 在左子树中找最大值，或在右子树中找最小值替代 |

### 6.7 中序后继

![中序后继|400](https://api2.mubu.com/v3/document_image/ece2516a-e5a8-4a36-aee8-58001158f6c8-28368310.jpg)

| 情况 | 中序后继 |
|------|----------|
| 节点在左子树，父节点有右子树 | 右子树的最深处（最左端） |
| 节点在左子树，父节点无右子树 | 父节点 |
| 节点在右子树 | 祖父节点（与上一情况等价） |

## 7.图

### 7.1 基本概念

图由顶点集合 V 和边集合 E 组成。

- **有序对**：$(a, b) \ne (b, a)$
- **无序对**：$(a, b) = (b, a)$

![顶点和边的表示|400](https://api2.mubu.com/v3/document_image/54290544-6816-4610-9865-9e50e9becc96-28368310.jpg)

| 类型 | 定义 |
|------|------|
| 有向图（Digraph） | 所有边都是单向或双向的 |
| 无向图（Graph） | 所有边都是无向的 |

### 7.2 基本属性

| 属性 | 说明 |
|------|------|
| 权重 | 每条边的成本不同 |
| 自环 | 自己指向自己 |
| 多重边 | 两点之间允许多条边 |

**边数**（顶点数 $V = n$）：

| 图类型 | 最大边数 |
|--------|----------|
| 有向图 | $E \le n(n-1)$ |
| 无向图 | $E \le \dfrac{n(n-1)}{2}$ |

根据边数可判断图的稀疏程度。

**路径**：

| 类型 | 定义 |
|------|------|
| Walk（途径） | 相邻点通过边连接，点和边允许重复 |
| Trail（迹） | 点可重复，边不可重复 |
| Simple Path（简单路径） | 点和边均不重复（最有意义，一般讨论的路径即指此） |

> 如果存在一条路径，则必包含一条简单路径。

**连接性**：

| 概念 | 定义 |
|------|------|
| 强连接 | 有向图中，从一个点可以到达任意其他点 |
| 连接 | 无向图中，所有点都被至少一条边连接 |

**环**：

| 类型 | 定义 |
|------|------|
| Closed Walk | 从原点出发又返回原点，允许重复 |
| Simple Cycle（环） | 从原点出发又返回原点，不许重复 |
| 无环图 | 没有循环 |

### 7.3 图的表示法

> 在大多数情况下，引用的成本比拷贝低很多。

最频繁的操作：① 寻找一个节点的所有相邻点；② 寻找两个节点间的路径。

#### 7.3.1 边列表

![边列表|400](https://api2.mubu.com/v3/document_image/90b2bcaf-7101-4547-b998-bc2ad3276f29-28368310.jpg)

| 维度 | 成本 |
|------|------|
| 空间 | O(V + E) |
| 找相邻点（最坏） | O(E) |
| 找两点路径 | O(V²) |

#### 7.3.2 邻接矩阵

![邻接矩阵|400](https://api2.mubu.com/v3/document_image/0c1a89a3-7b10-43a4-8803-d3c178e42e99-28368310.jpg)

![实际成本|400](https://api2.mubu.com/v3/document_image/78672299-0504-4135-941c-c708bbfeed69-28368310.jpg)

| 维度 | 成本 |
|------|------|
| 空间 | O(V²) |
| 找相邻点（最坏） | O(V) |
| 找两点路径 | O(V) |

> [!warning] 缺点
> 矩阵消耗 O(V²) 空间，对稀疏图造成巨大浪费，不适合大型稀疏图。

#### 7.3.3 邻接表

![邻接表|400](https://api2.mubu.com/v3/document_image/ce172305-dced-40a0-9e5d-286bf9e1d291-28368310.jpg)

每个节点单独分配一个链表，用指针数组存储各链表头节点地址：

![邻接表结构1|400](https://api2.mubu.com/v3/document_image/748a7c89-2919-4fe6-9bdb-df72a6ab2bee-28368310.jpg)

![邻接表结构2|400](https://api2.mubu.com/v3/document_image/3b54f80d-163b-4bdb-b5fe-bc9836560fff-28368310.jpg)

| 维度 | 成本 |
|------|------|
| 空间 | O(E)，且 $E \ll V^2$（去除了大量 0） |
| 找相邻点（最坏） | O(V) |
| 找两点路径 | O(V) |

> 适用于稀疏图。

#### 7.3.4 三种表示法对比

| 维度 | 边列表 | 邻接矩阵 | 邻接表 |
|------|--------|----------|--------|
| 空间 | O(V + E) | O(V²) | O(E) |
| 找相邻点（最坏） | O(E) | O(V) | O(V) |
| 找两点路径 | O(V²) | O(V) | O(V) |
| 适用场景 | 边数少 | 稠密图 | 稀疏图 |
