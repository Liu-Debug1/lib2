# 1. 新手入门（New Hand）
### 1.1 搭建第一个Keil项目

操作流程：创建空项目 → 新建项目文件夹 → 源文件（右键 → 添加 → 新建`.c`文件）→ 编写代码
> [!warning] 代码编写注意事项
> 1. 头文件调用规则：调用标准C库用`#include <xxx.h>`，调用自定义头文件用`#include "xxx.h"`
> 2. `main`函数是程序唯一入口，一个工程有且仅有1个`main`函数
> 3. 每个语句必须以英文分号`;`结尾
> 4. 函数的开始和结尾必须用大括号`{}`包裹

### 1.2 注释语法
- 单行注释：`// 注释信息`
- 多行注释：`/* 注释内容 */`，注释嵌套时仅匹配最近的`*/`结束符
- Keil快捷键：
  - 添加注释：`Ctrl+K` + `Ctrl+C` 组合键
  - 取消注释：`Ctrl+K` + `Ctrl+U` 组合键

### 1.3 关键字
C语言中具有特定含义的英文单词，特点：
1. 全部小写
2. 编译器会自带高亮颜色（蓝/紫色）
3. 不能用作变量名/函数名

### 1.4 位、字节、字与进制

#### 一、基本单位

计算机里最小的三个存储单位：

| 单位  | 英文         | 大小         | 说明                            |
| --- | ---------- | ---------- | ----------------------------- |
| 位   | bit（b，小写）  | 1 个 0 或 1  | 最小单位，一个开关                     |
| 字节  | Byte（B，大写） | 8 bit      | C 语言里 `char` 的大小              |
| 字   | Word       | 取决于 CPU 位宽 | 32 位机 = 4 Byte，64 位机 = 8 Byte |

> [!NOTE] 约定：说"位"默认指二进制位
> "32 位 CPU""64 位电脑"里的"位"都是二进制位。不提进制时，"位" = bit = 一个二进制位。

> [!tip] 类比
> bit = 一个灯泡（亮/灭），Byte = 8 个灯泡排一排（一个"格子"），Word = CPU 一次能读的一整排格子。

核心换算关系（必须记住）：

```
1 Byte = 8 bit = 2 位十六进制
1 Word = 4 Byte = 32 bit = 8 位十六进制（32 位 CPU）
```

> [!tip] 为什么 1 Byte = 2 位十六进制？
> 1 位十六进制 = 4 bit（2⁴=16），1 Byte = 8 bit，所以刚好是 2 位。`0xFF` = `0b11111111` = 255 = 一个字节的最大值。

#### 二、四种进制

同一个数值，可以用不同的进制表示——就像"十块钱"和"10 元"表达的是同一个金额，只是写法不同。

| 进制 | 基数 | 用到的符号 | C 语言前缀 | 例子 |
|------|------|-----------|-----------|------|
| 二进制 | 2 | 0、1 | `0b` | `0b1010` = 10 |
| 八进制 | 8 | 0~7 | `0` | `012` = 10 |
| 十进制 | 10 | 0~9 | 无 | `10` |
| 十六进制 | 16 | 0~9、A~F | `0x` | `0xA` = 10 |

#### 三、进制之间的关系——为什么刚好是这些进制

核心原因是 **8 和 16 都是 2 的幂**，和二进制之间有"整截"的关系：

```
十六进制 1 位 = 二进制 4 位（2⁴ = 16）
八进制   1 位 = 二进制 3 位（2³ = 8）
```

这就是为什么嵌入式里最爱用十六进制——一个 `uint32_t` 刚好是 8 位十六进制数，寄存器值一目了然：

```
二进制：  0b 1100 1010 1111 0000 0101 1010 0010 1100
十六进制：0x C    A    F    0    5    A    2    C
```

如果写成十进制 `3405775404`，完全看不出每一位的含义。但 `0xCAF05A2C` 可以直观地拆成 4 个字节：`CA` `F0` `5A` `2C`。

> [!tip] 八进制有什么用？
> 八进制主要用于 Linux 文件权限（`chmod 755`），嵌入式开发中不常用。

#### 四、不同进制之间的转换

**任意进制 → 十进制**：按权展开求和

```
0x2F = 2×16¹ + 15×16⁰ = 32 + 15 = 47
0b1101 = 1×2³ + 1×2² + 0×2¹ + 1×2⁰ = 8 + 4 + 0 + 1 = 13
```

**十进制 → 任意进制**：除基取余，倒序排列

```
47 转十六进制：
47 ÷ 16 = 2 余 15(F) → 2 ÷ 16 = 0 余 2 → 结果：0x2F
```

**二进制 ↔ 十六进制**：四位一组直接译

```
0b 1011 0110
   ↓    ↓
   B    6    → 0xB6
```

**二进制 ↔ 八进制**：三位一组直接译

```
0b 10 110 110
   ↓  ↓   ↓
   2  6   6   → 0266
```

#### 五、一张图总结关系

```text
           ┌──────────────────────────────────┐
           │         1 Word（32 位 CPU）       │
           │         = 4 Byte = 32 bit        │
           │                                  │
           │  ┌──────────┬──────────┬──────────┬──────────┐
           │  │ Byte 3   │ Byte 2   │ Byte 1   │ Byte 0   │
           │  │ 8 bit    │ 8 bit    │ 8 bit    │ 8 bit    │
           │  │          │          │          │          │
           │  │ 0x CA    │ 0x F0    │ 0x 5A    │ 0x 2C    │ ← 十六进制
           │  │          │          │          │          │
           │  │ 0b 1100 1010 │ 0b 1111 0000 │ 0b 0101 1010 │ 0b 0010 1100 │ ← 二进制
           │  └──────────┴──────────┴──────────┴──────────┘
           └──────────────────────────────────┘
```

> [!NOTE] 嵌入式开发中
> - 寄存器配置用**十六进制**（`0x` 前缀）
> - 位操作（置位、清零）用**二进制**思维（第几位是 1）
> - 十进制只在算数值大小时用
> - 八进制几乎不出现

# 2. 核心语法
### 2.1 数据存储规则
所有数据最终都以二进制形式存储在内存中：

| 数据类型 | 存储逻辑 |
| --- | --- |
| 文本 | 汉字 → GBK编码（十进制）→ 二进制；英文/数字 → ASCII编码（十进制）→ 二进制 |
| 图像 | 按分辨率拆分像素，每个像素的RGB三色值分别编码为二进制存储 |
| 声音 | 按采样频率采集幅值，每个幅值编码为二进制存储；采样频率越高还原度越高，存储占用空间越大 |

### 2.2 格式化输出`printf`

语法：`printf("格式化字符串", 参数列表);`
> 要求：占位符的类型、顺序必须和参数一一对应
#### 常用占位符）
| 占位符       | 对应类型    | 备注                                 |
| --------- | ------- | ---------------------------------- |
| `%d`      | 整型      | 十进制输出                              |
| `%f`      | 实型（浮点数） | 小数输出                               |
| `x%`/`X%` | 十六进制    | [注：标准写法为`%x`（小写十六进制）/`%X`（大写十六进制）] |
| `o%`      | 八进制     | [注：标准写法为`%o`]                      |
| `%c`      | 字符型     | 单个字符输出                             |
| `%s`      | 字符串     | 字符串输出                              |
#### 换行符
- Windows系统：`\r\n`，直接用`\n`也可兼容
- Mac系统：`\r`
- Linux系统：`\n`

### 2.3 变量
变量是用于存储数据的容器，可赋值、运算、修改。

> [!warning] 变量使用注意事项
> a. 一个变量只能存储一个数据
> b. 同一作用域内变量名不可重复
> c. 支持一次性定义多个变量（工程中不推荐，可读性差）
> e. 变量未赋值前无法使用，值为随机数
> f. 变量有作用范围限制
#### 静态变量（`static`修饰）
静态变量的生命周期贯穿整个程序运行期，按定义位置分为两类：

| 类型 | 作用域 | 特性 |
| --- | --- | --- |
| 全局静态变量 | 全局作用域定义 | 可在当前文件定义之后的任意位置访问 [工程补充：全局静态变量作用域仅限当前`.c`文件，无法通过`extern`跨文件访问，如需跨文件访问不要加`static`修饰] |
| 局部静态变量 | 函数内部定义 | 仅能在定义它的函数内部访问，仅在程序启动时初始化1次，函数退出后值不会丢失，下次调用函数时保留上次的值 |

### 2.4 常量
程序运行过程中值不会发生变化的数据，分类：

| 常量类型 | 定义规则 | 示例 |
| --- | --- | --- |
| 整型 | 整数 | `103`、`-403`、`0` |
| 实型 | 带小数点的数 | `1.23`、`1e3`（科学计数法） |
| 字符型 | 英文单引号包裹，单个字符/英文符号/空格 | `'a'`、`'1'`、`' '` |
| 字符串型 | 英文双引号包裹，内容不限 | `"勤奋好学威"`、`""`（空字符串） |

### 2.5 数据类型
| 类型                        | 长度（常见平台）          | 取值范围                                        | 说明                                                                                             |
| ------------------------- | ----------------- | ------------------------------------------- | ---------------------------------------------------------------------------------------------- |
| `int`（有符号整型）              | 4字节（32位）          | `-2^31`(-2147483648) ~ `2^31-1`(2147483647) | 最高位为符号位，0正1负                                                                                   |
| `unsigned int`（无符号整型）     | 4字节（32位）          | 0 ~ `2^32-1`(4294967295)                    | 所有位都用于表示数值                                                                                     |
| `short`（短整型）              | 2字节（16位）          | 有符号：-32768~32767；无符号：0~65535                | 用于存储空间受限的场景                                                                                    |
| `long`（长整型）               | 32位系统4字节/64位系统8字节 | 同对应位数的整型                                    | 32位系统与int范围一致                                                                                  |
| `long long`（C99引入）        | 8字节（64位）          | 有符号：-9e18~9e18；无符号：0~1.8e19                 | 用于存储极大整数                                                                                       |
| `char`（字符型）               | 1字节（8位）           | 有符号：-128~127；无符号：0~255                      | 可存储字符也可存储8位整数，赋值的十六进制数会自动转二进制存储                                                                |
| `bool`（布尔型，C99引入）         | 1字节               | `true`(1) / `false`(0)                      | 需要包含头文件`<stdbool.h>`                                                                           |
| 固定宽度整型（C99`<stdint.h>`定义） | 8/16/32/64位       | 对应位数的整数范围                                   | 常用`int8_t`/`uint8_t`/`int16_t`/`uint32_t`等，**[车规开发补充：MISRA规范要求优先使用固定宽度整型，避免不同平台位数差异导致的兼容性问题]** |
|                           |                   |                                             |                                                                                                |

#### 浮点型
浮点型采用 **IEEE 754** 标准，由三部分组成：符号位、指数、尾数。小数点位置由指数动态决定（所以叫「浮点」），因此能用有限位数表示极大或极小的数。

| 类型            | 总位数      | 符号  | 指数   | 尾数   | 有效精度（十进制） | 取值范围         |
| ------------- | -------- | --- | ---- | ---- | --------- | ------------ |
| `float`       | 32 位     | 1 位 | 8 位  | 23 位 | 6~7 位     | ±3.4 × 10³⁸  |
| `double`      | 64 位     | 1 位 | 11 位 | 52 位 | 15~16 位   | ±1.8 × 10³⁰⁸ |
| `long double` | 80/128 位 | —   | —    | —    | 看编译器      | 看编译器         |

> [!warning] 浮点型注意事项
> 1. **整数精确表示有限**：`float` 精确表示约 1677 万以内的整数（8 位以内安全），`double` 约 9007 万亿（16 位以内安全）。超过这个范围整数会丢失低位。
> 2. **越大的数越稀疏**：数值越大，相邻可表示数之间的间隔越大，中间的值被跳过。
> 3. **默认用 `double`**：日常编程中 `double` 精度足够，`float` 只适用于内存敏感场景（如嵌入式大量传感器数据）。
 

> 补充：寄存器本身不分「整数位」和「小数位」，定点数的小数点位置由程序员约定（如嵌入式 Q 格式定点数），浮点数则通过指数自动浮动。

### 2.6 进制转换与存储
- 存储逻辑：所有数值（十进制/十六进制/八进制）赋值给变量后，都会自动转换为二进制存储在内存中，进制只是人类可读的不同表示形式
- 输出逻辑：从内存读取变量时，默认转十进制显示，可通过占位符指定为十六进制/八进制输出
- 通用流程：输入（任意进制/字符）→ 编码 → 二进制存储 → 译码 → 输出（指定编码格式）

# 3. 流程控制语句
### 3.1 顺序结构
代码从上到下依次执行，为程序默认执行流程。
### 3.2 分支结构
用于满足特定条件才执行对应逻辑的场景：
#### `if`语句
支持三种结构：
1. 单分支：仅满足条件时执行
2. 双分支：二选一执行
3. 多分支：从上到下遍历条件，满足第一个条件后执行并退出，不再判断后续条件
#### `switch`语句
适用于有限个固定值匹配的场景，把所有条件一一列举，匹配对应分支执行。
#### `if`与`switch`选择规则
1. 范围匹配类场景用`if`
2. 有限个固定值匹配类场景用`switch`，执行效率高于`if`
### 3.3 循环结构
用于重复执行某段代码的场景，支持`while`、`do-while`、`for`三种循环。

#### `for` vs `while` 选择原则

**结论：遍历已知次数的场景用 `for`，更合适。**

| 对比维度 | `for` | `while` |
|---------|-------|---------|
| 循环变量作用域 | 限定在循环体内，用完销毁 | 定义在循环外部，结束后仍存活 |
| 三要素位置 | 初始化、条件、步进集中在一行 | 三要素分散在各处 |
| 适用场景 | 已知遍历次数 | 未知循环次数（等待条件成立） |

**作用域优势示例：**

```cpp
// while：i 在循环结束后还活着，污染外层作用域
int i = 0;
while (i < 10) {
    i++;
}
// i 仍然可用，但已无意义，且可能被误用

// for：i 生命周期绑定在循环内，作用域干净
for (int i = 0; i < 10; i++) {
    // 循环体
}
// i 已销毁，外面访问不到
```

#### 循环变量同名遮蔽

如果函数内已定义了 `int i`，`for` 头部仍可以声明 `int i`，内层会临时遮蔽外层的同名变量：

```cpp
int i = 5;                     // 外层 i

for (int i = 0; i < 10; i++)   // 声明自己的 i，遮蔽外层
{
    cout << i;                 // 输出 0~9，内层 i
}

cout << i;                     // 输出 5，外层 i 未被修改
```

> [!warning] 同名遮蔽虽然合法，但容易造成混淆。实践中建议循环变量换名（如 `idx`、`j`），避免歧义。

# 4. 宏定义 & 预处理

当程序中有多个地方运用同一个标识符时且在未来可能需要更改，则可以进行宏定义统一调度。
即避免修改时出现差错，又可以做到见名知意。

## 4.1 常见指令

- `#include` —— 文件包含
- 条件编译块（`#ifdef`、`#ifndef`、`#else`、`#endif`）
- `#define` & `#undef` —— 宏定义与取消
- 宏的要点（见下文）

## 4.2 宏

### 1. 基本语法

- **无参数**：`#define 宏名 替换内容`
- **带参数**：`#define 宏名(参数列表) 替换内容`

### 2. 常用技巧

- **技巧1：加括号**
  说明：给参数和整个表达式加括号，防止运算符优先级错误。
  示例：`#define MUL(x,y) ((x)*(y))`

- **技巧2：do-while封装**
  说明：用 `do{...}while(0)` 封装多语句宏，使其像函数一样使用。
  示例：`#define DELAY_MS(ms) do{ ... }while(0)`
  > 注意：`\` 是换行符，表示下一行与当前行属于同一宏定义。

- **技巧3：位操作宏**
  说明：定义位设置、位清除、位读取等常见操作。
  示例：`#define SET_BIT(reg,bit) ((reg) |= (1<<(bit)))`

- **技巧4：`##` 连接符**
  说明：连接宏定义的变量，可用于批量定义函数与变量。
  示例：`#define CONCAT(a,b) a##b`

```c
// 定义一个通用的引脚操作宏
#define GPIO_WRITE(port, pin, val)  GPIO_WriteBit(GPIO##port, GPIO_Pin_##pin, (BitAction)(val))
// 调用示例：
GPIO_WRITE(B, 8, 1);
// 预处理器会把 GPIO##port 变成 GPIOB，GPIO_Pin_##pin 变成 GPIO_Pin_8
// 替换后实际执行的库函数：GPIO_WriteBit(GPIOB, GPIO_Pin_8, (BitAction)(1));
```

- **技巧5：`#` 字符串化**
  说明：将宏参数转变为字符串。
  示例：`#define TO_STR(x) #x`

> **补充说明**：`\` 是换行符，表示下一行与当前行属于同一宏定义。
> 例如：`#define DELAY_MS(ms)`  下一行 `do{ ... }while(0)` 前加 `\` 即可。

# 6. 枚举 enum

- 定义与结构体相似，只需将 `struct` 替换为 `enum`。
- **限制变量取值**：枚举变量只能取枚举内部定义的成员值，防止赋值错误。
- **默认值**：第一个枚举成员默认为 0，后续依次递增 1。
- **手动赋值**：可以显式指定整数值，打破默认递增顺序。

示例：
```c
enum Weekday {
    MON = 1,   // 指定 1
    TUE,       // 2
    WED,       // 3
    THU,       // 4
    FRI,       // 5
    SAT = 10,  // 指定 10
    SUN        // 11 (接在10后面)
};
```

> **补充说明**：枚举常用于状态机中的状态定义（如车窗控制中的 `IDLE`、`MOVING_UP`、`MOVING_DOWN`、`FAULT` 等），清晰且安全。


# 7. 指针

## 7.1 计算机存储机制

- 每个字节有自己**独一无二的地址**
- **字节序**：`int a = 0x12312` — 小端模式低位在前，大端模式低位在后
- **数组存储**：`char c[] = {0x11, 0x22, 0x21}` — 数据按顺序依次存储

![内存存储示意图\|400](https://api2.mubu.com/v3/document_image/fe6494fc-c19b-48b0-8587-cf5bbe36d866-28368310.jpg?)

## 7.2 指针基础

### 定义

> [!info] 指针 (Pointer)
> 使用指针可操作数据的**地址**，实现数据的**间接访问**。

| 语法 | 含义 | 示例 |
|:-----|:-----|:-----|
| `int *p` | 声明一个指向 `int` 的指针变量 | `int *p = &a;` |
| `&a` | 取变量 `a` 的地址 | `p = &a;` |
| `*p` | 解引用，取出指针指向的值 | `int val = *p;` |

![指针定义语法\|570](https://api2.mubu.com/v3/document_image/b01d4954-ec0c-4573-a33e-2db0cb0804cb-28368310.jpg?)

### 指针变量

- 指针变量**存储**数据单元的首地址 → 该指针指向了某个数据单元
- 未指向任何数据单元 → **空指针** (`NULL`)

### 指针操作

| 操作      | 含义       | 说明                      |
| :------ | :------- | :---------------------- |
| `*p`    | 解引用      | 读取/修改指针指向的值             |
| `p++`   | 指针偏移     | 增加一个数据类型存储单位的宽度，常用于数组遍历 |
| `p--`   | 指针回退     | 减少一个单位                  |
| `p + n` | 偏移 n 个单位 | 不改变 p 本身                |

> [!warning] 注意
> - 指针的**乘除运算没有实际意义**
> - `p++` 要避免越界，一般用在数组内部

![指针操作示意图\|801](https://api2.mubu.com/v3/document_image/4b3fa34f-1163-4dfa-8069-4e1171a38be0-28368310.jpg?)

### `->` 操作符

`pPerson->name` 等价于 `(*pPerson).name`：
1. 先解引用 `pPerson` 得到结构体
2. 再用 `.` 访问成员

`->` 是一种更简洁的写法，专用于**指针访问结构体成员**。

## 7.3 数组与指针

| 表达式 | 含义 |
|:-------|:-----|
| `c` / `&c[0]` | 数组首元素地址 |
| `*c` / `c[0]` / `*p` / `p[0]` | 数组第一个元素的值 |

> [!tip] 核心关系
> **数组是指针的另一种形态**——数组名可视为指向首元素的指针常量。数组和指针在函数传参时可以混用。

![数组与指针关系\|794](https://api2.mubu.com/v3/document_image/2508a3b1-2d43-43a9-928d-c54ffb5be3d4-28368310.jpg?)

## 7.4 指针级别转换

| 操作      | 输入   | 输出   | 规则             |
| :------ | :--- | :--- | :------------- |
| `&`（升级） | 变量   | 一级指针 | `&变量` → 一级指针   |
| `&`（升级） | 一级指针 | 二级指针 | `&一级指针` → 二级指针 |
| `*`（降级） | 一级指针 | 变量   | `*一级指针` → 变量   |
| `*`（降级） | 二级指针 | 一级指针 | `*二级指针` → 一级指针 |

> [!important] 核心规则
> **同级指针之间才可以相互赋值。**
> - 跨级赋值：低级指针需要用到取地址符 `&`
> - 同级赋值：`a = p`（两者都是同级指针）

## 7.5 指针应用

### 传递参数

**a. 传递地址，避免大量空间浪费**

| 传值方式                | 行为                   | 内存开销       |
| :------------------ | :------------------- | :--------- |
| 传值 `func(int data)` | 函数创建局部变量，**复制**整个数据  | 与数据等大      |
| 传址 `func(int *p)`   | 函数创建指针变量，存储源数据**地址** | 仅 8 字节（指针） |

> [!tip] 最佳实践
> - 大数据结构（如结构体）传参 → 用指针，节省栈空间
> - `const` 修饰指针形参 → 只读访问，防止误改源数据

**b. 返回多个值**

函数只能 return 一个值。通过指针参数直接修改源数据，实现"返回多个值"的效果：

```c
void GetMinMax(int arr[], int n, int *min, int *max) {
    *min = arr[0];
    *max = arr[0];
    for (int i = 1; i < n; i++) {
        if (arr[i] < *min) *min = arr[i];
        if (arr[i] > *max) *max = arr[i];
    }
}
// 调用：GetMinMax(data, 10, &minVal, &maxVal);
```

### 传递返回值

函数可返回指针，让调用者直接访问函数内部动态分配或静态存储的数据。

### 直接访问硬件地址

嵌入式开发中，指针可直接操作**外设寄存器**：

```c
#define GPIOB_ODR  (*(volatile uint32_t *)0x40010C0C)
GPIOB_ODR |= (1 << 8);  // 直接操作 GPIOB 引脚 8 输出高电平
```

## 7.6 注意事项

> [!warning] 使用指针前务必检查
> 1. **解引用前确保指针指向合法地址**，否则产生段错误/硬件异常
> 2. **避免野指针**：指针变量定义后立即初始化（指向有效地址或置 `NULL`），释放动态内存后将指针置 `NULL`
> 3. **同级赋值**：不同级别的指针之间不能直接赋值，需用 `&` 或 `*` 转换级别

## 7.7 函数指针

### 定义

函数指针是一个**指向函数的指针变量**——代码本身存在内存里，函数指针存的就是那段代码的入口地址。

| 语法 | 含义 | 示例 |
|:-----|:-----|:-----|
| `int (*p)(int, int)` | 声明 p 指向 "返回 int、接收两个 int" 的函数 | `int (*p)(int, int) = add;` |
| `p = func` | 让 p 指向 func（`&` 可省略） | `p = add;` 等价于 `p = &add;` |
| `p(3, 5)` | 通过指针调用函数（`*` 可省略） | `p(3, 5);` 等价于 `(*p)(3, 5);` |

> [!warning] 括号不能省
> `int (*p)(int)` 是函数指针，`int *p(int)` 是返回 `int*` 的函数声明，两者完全不同。

### 为什么需要函数指针

普通指针存数据，函数指针**存行为**——它让你在运行时决定"调用哪个函数"，而不是在编译时就写死。

### 常见用法

**用法 1：回调函数——把函数当参数传**

```c
// qsort 标准库就是函数指针的经典用法
int compare(const void* a, const void* b) {
    return *(int*)a - *(int*)b;
}

int arr[] = {3, 1, 4, 1, 5};
qsort(arr, 5, sizeof(int), compare);  // 把 compare 传给 qsort
```

**用法 2：状态机——根据条件切换执行不同函数**

```c
typedef void (*StateFunc)(void);

void state_idle(void)    { /* 空闲处理 */ }
void state_running(void) { /* 运行处理 */ }
void state_error(void)   { /* 故障处理 */ }

StateFunc current_state = state_idle;
// 切换状态就是换个指针
current_state = state_running;
current_state();  // 执行当前状态逻辑
```

**用法 3：命令表——用索引选函数（嵌入式常用）**

```c
typedef void (*CmdFunc)(void);

void cmd_start(void) { ... }
void cmd_stop(void)  { ... }
void cmd_reset(void) { ... }

CmdFunc cmd_table[] = { cmd_start, cmd_stop, cmd_reset };
cmd_table[1]();  // 收到命令字 1，执行 cmd_stop
```

> [!tip] typedef 简化类型名
> 函数指针的原始语法很长，用 `typedef` 给类型起个短名：`typedef int (*Operation)(int, int);`，之后 `Operation p = add;` 就干净多了。

### 完整示例：计算器

```c
#include <stdio.h>

typedef int (*Operation)(int, int);

int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }
int mul(int a, int b) { return a * b; }

int calculate(int x, int y, Operation op) {
    return op(x, y);
}

int main(void) {
    printf("%d\n", calculate(10, 5, add));  // 15
    printf("%d\n", calculate(10, 5, sub));  // 5
    printf("%d\n", calculate(10, 5, mul));  // 50
    return 0;
}
```

### 函数指针 vs 普通指针

| | 普通指针 | 函数指针 |
|:---|:---|:---|
| 指向谁 | 数据 | 代码（函数入口） |
| 取地址 | `&变量` | `函数名` 或 `&函数名` |
| 解引用 | `*p` 读写数据 | `p()` 调用函数 |
| 偏移运算 | `p++` / `p--` 有意义（数组遍历） | **无意义**——函数不等长 |
| 跨级转换 | `&p` → 二级指针，`*p` → 降级 | 无级别概念 |

### 嵌入式中的应用

裸机 C 开发中函数指针很实用，三个最常见的场景：

| 场景 | 说明 |
|:-----|:-----|
| 中断回调 | 把用户处理函数注册到驱动层，中断触发时回调 |
| 外设驱动表 | 用结构体 + 函数指针做 HAL 抽象，统一 SPI/I2C/UART 接口 |
| 裸机状态机 | 用函数指针数组实现状态跳转，比 `switch-case` 更好维护 |

> [!NOTE] C 语言 vs C++
> 以上是 C 语言中的函数指针。在 C++ 中，函数指针还有一种特殊形式——**成员函数指针**（`void (Dog::*pb)() = &Dog::bark;`），以及更现代的 `std::function` + lambda 替代方案。这些属于 C++ 范畴，详见 C++ 相关笔记。
