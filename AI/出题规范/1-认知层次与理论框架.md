# 认知层次与理论框架

> 来源：Bloom 分类学、SOLO 分类理论、《教育考试的理论与方法》(王后雄)、《教育测量与评价》(黄光扬)

---

## 一、Bloom 认知分类学（六级）

| 层级 | 含义 | 题干关键词（Question Stems） | C/C++ 举例 |
|------|------|---------------------------|-----------|
| 1. 记忆 | 回忆事实/概念/规则 | 列出、定义、写出、说出 | "const 关键字的作用是什么？" |
| 2. 理解 | 用自己的话解释、识别、举例 | 解释、举例、分类、简述 | "简述引用和指针的区别" |
| 3. 应用 | 在给定情境中使用知识 | 使用、计算、实现、编写 | "用 new/delete 实现动态数组" |
| 4. 分析 | 分解、比较、找出关系 | 比较、分析、找出错误 | "指出以下代码中的内存泄漏" |
| 5. 评价 | 基于标准判断、论证 | 判断、评价、论证、选择最优 | "给定3种实现方案，论证最优方案" |
| 6. 创造 | 综合、设计、产生新方法 | 设计、构建、提出、重新组织 | "设计一个线程安全的单例模式" |

**关键结论**：多数工程考试题目停留在记忆/理解层。好的出题应确保记忆层 < 30%，应用、分析、评价层占主体。

## 二、SOLO 分类理论（五个结构层次）

| 层次 | 说明 | 示例 |
|------|------|------|
| 前结构 | 完全不理解，乱答 | — |
| 单点结构 | 只能用单一知识点回答 | "这是什么类型？int a = 5" |
| 多点结构 | 列举多个独立知识点 | "C++ 有哪几种初始化方式？" |
| 关联结构 | 将多个知识点综合关联分析 | "指针和数组的相互转换与应用，结合 const" |
| 扩展抽象 | 迁移到新场景，提出新解法 | "将状态机模式应用到 CAN 通信协议解析" |

**题目难度与层级对应**：基础题 = 单点/多点，中等题 = 关联，难题 = 扩展抽象

## 三、难度分布建议

| 难度 | 比例 | 目标 |
|------|------|------|
| 易 | 20-30% | 巩固基础 |
| 中 | 30-40% | 提升应用 |
| 难 | 20-30% | 区分拔高 |

整卷难度系数建议控制在 0.5 左右（平均正确率 50%）。

## 四、双向细目表（命题施工图）

命题前必须编制！横轴 = 认知层次，纵轴 = 知识点。

| 知识点 | 记忆 | 理解 | 应用 | 分析 | 评价 | 合计 |
|--------|------|------|------|------|------|------|
| 指针 | 1 | 1 | 2 | 1 | 0 | 5 |
| class | 0 | 1 | 2 | 1 | 1 | 5 |
| ... | | | | | | |

**作用**：确保每个认知层次都有覆盖，没有偏废某类题目。
